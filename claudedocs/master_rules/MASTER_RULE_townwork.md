## 🔴 絶対ルール（有料媒体共通）

1. **成約先除外必須** - 成約先に営業架電すると事故
2. **電話済み除外** - 媒体掲載中リスト内の電話番号は除外
3. **ユーザー確認必須** - インポート前に件数・サンプル提示
4. **バックアップ取得** - 更新実行前に対象レコードのバックアップ

※詳細は `MASTER_RULE_paid_media.md` を参照

## 🔴 タウンワーク固有ルール

### 重複マージ処理

タウンワークデータは同一電話番号で複数求人が存在するため、マージ処理が必須。

```
処理フロー:
1. 電話番号で重複をカウント
2. 同一電話番号のレコードをマージ（最初のレコードを採用）
3. マージ数を duplicate_count 列に記録
4. マージされた職種を merged_job_titles 列に記録
```

**マージ数の活用**:
- `Paid_NumberOfRecruitment__c` にマージ数を格納
- メモ欄に「マージ数: X件」を記載
- 求人数が多い会社 = 採用意欲が高い可能性

## 処理フロー

```
[STEP 1] データ読み込み
├─ タウンワークCSV読み込み（dtype=str必須）
└─ 電話番号正規化

[STEP 2] 重複マージ
├─ 電話番号でグループ化
├─ duplicate_count算出
└─ merged_job_titles生成

[STEP 3] 除外処理
├─ 成約先電話番号除外
├─ 電話済み電話番号除外
└─ 対象外業種除外 ← タウンワーク固有

[STEP 4] Salesforce突合
├─ Lead電話番号マッチ → Lead更新
├─ Account電話番号マッチ → Account更新
└─ マッチなし → 新規リード

[STEP 5] CSV生成
├─ import_new_leads_YYYYMMDD.csv
├─ import_lead_updates_YYYYMMDD.csv
└─ import_account_updates_YYYYMMDD.csv

[STEP 6] インポート（ユーザー確認後）
```

## 注意事項

### 電話番号の形式
- 元データに11桁で`00`始まりの番号がある場合、先頭の`0`を1つ削除
- 携帯番号（070/080/090）は`Phone`と`MobilePhone`の両方に設定（Phone必須のため）

### 担当者名の処理
- 代表者名があれば`LastName`に設定
- なければ「担当者」を設定（必須フィールド）

### メモ欄（LeadSourceMemo__c）
```
【新規作成】タウンワーク突合 YYYY-MM-DD
マージ数: X件
職種: [マージされた職種一覧]
```
