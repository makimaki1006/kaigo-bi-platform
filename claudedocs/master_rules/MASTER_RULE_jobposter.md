## ジョブポスター固有の特徴

### データソースの特殊性

| 項目 | 特徴 | 他媒体との違い |
|-----|------|--------------|
| ファイル形式 | Excel (.xlsx) | ミイダスはCSV |
| 列数 | 約10列（最少） | ミイダスは278列、PT・OT・STは約20列 |
| 電話番号の位置 | 専用列（単一） | PT・OT・STは複数列（電話番号_1〜10） |
| 職種 | 職 種列（専用） | ミイダスは全本文ダンプから抽出 |
| ウェブサイト | ホームページ列あり | PT・OT・STネットはなし |
| 掲載期間 | あり | 他媒体は掲載日のみ |
| 代表者情報 | **なし** | ミイダスのみあり |
| 業界分類 | **なし** | PT・OT・STはリハビリ分類 |
| 雇用形態 | **なし** | PT・OT・STはあり |
| 採用人数 | **なし** | PT・OT・STはあり |

| 業界 | 職種例 |

## データソース仕様

### ファイル形式
- **ファイル名**: `ジョブポスター.xlsx`
- **形式**: Excel (.xlsx)
- **エンコーディング**: UTF-8
- **シンプルな構造**: 約10列のみ

### 列構成（2026年1月時点）

| 列名 | 説明 | 必須 | 備考 |
|-----|------|------|------|
| URL | 求人ページURL | Yes | |
| 職 種 | 職種名（介護職、看護師等） | Yes | スペースを含む列名に注意 |
| 勤務地 | 住所（都道府県から） | Yes | |
| 会社情報 | 会社名（改行で詳細が続く場合あり） | Yes | 1行目のみ使用 |
| ホームページ | 会社ウェブサイト | No | Websiteにマッピング |
| 応募担当者名 | 担当者名 | No | |
| 電話番号 | 連絡先電話番号（1つ） | Yes | 単一列 |
| 掲載期間 | 掲載期間情報 | No | メモ欄に記載 |

### 電話番号の特徴

- **単一列**: 電話番号列は1つのみ
- 固定電話と携帯電話が混在
- 正規化後10〜11桁で判定

### 会社情報の特徴

```
株式会社サンプル
〒123-4567 東京都...
事業内容: ...
```

- 改行で会社名と詳細が分離されている場合あり
- **最初の行のみを会社名として使用**

## 会社名抽出ロジック

### 改行対応

```python
def extract_company_name(company_info: str) -> str:
    """会社情報から会社名を抽出（1行目のみ）"""
    if not company_info:
        return ""
    first_line = str(company_info).split('\n')[0].strip()
    return first_line if first_line else ""
```

### 注意点

- 会社情報に複数行の詳細が含まれることがある
- 住所情報は別途「勤務地」列を使用
- 会社名に株式会社・有限会社等が含まれる場合はそのまま使用

## セグメント分析

### ジョブポスターの制限

**ジョブポスターには「代表者」フィールドがないため、ミイダスのような「代表者直通」セグメント分析は不可。**

代替として以下の品質指標を使用：

| 指標 | 判定方法 | 優先度 |
|-----|---------|--------|
| 携帯電話あり | MobilePhoneに値あり | ★★★ |
| バイネームあり | LastNameが一般名称でない | ★★ |
| ウェブサイトあり | Websiteに値あり | ★ |

### 品質スコア計算

```python
def calculate_quality_score(row: pd.Series) -> int:
    score = 0
    if row.get('MobilePhone'):
        score += 3  # 携帯電話あり
    if row.get('LastName') and not is_generic_name(row['LastName']):
        score += 2  # バイネームあり
    if row.get('Website'):
        score += 1  # ウェブサイトあり
    return score
```

## 処理フロー

```
1. データ読み込み
   ├─ スクレイピングデータ（Excel）
   ├─ 成約先電話番号（CSV）
   ├─ 電話済み電話番号（Excel全シート）
   └─ Salesforce既存データ（Lead/Account/Contact）

2. 会社名抽出
   └─ 会社情報の1行目のみを抽出

3. 担当者名処理
   └─ 応募担当者名を使用（空なら「担当者」）

4. 除外処理
   ├─ 成約先電話番号一致 → 除外（理由: 成約先電話番号）
   └─ 電話済み電話番号一致 → 除外（理由: 電話済み）

5. 突合処理
   ├─ 電話番号で既存レコード検索
   ├─ マッチあり → 更新対象
   └─ マッチなし → 新規リード候補

6. 品質スコア計算
   ├─ 携帯電話有無
   ├─ バイネーム有無
   └─ ウェブサイト有無

7. CSV生成
   ├─ new_leads_final_*.csv（新規リード作成用）
   ├─ lead_updates_final_*.csv（Lead更新用）
   ├─ account_updates_final_*.csv（Account更新用）
   └─ excluded_final_*.csv（除外リスト）

8. 所有者割り当て
   └─ MASTER_RULE_paid_media.md §所有者割り当てルール参照

9. Salesforceインポート（ユーザー確認後）

10. メモ欄差別化
    └─ MASTER_RULE_paid_media.md §メモ欄差別化参照
```

## データ品質の特徴

### 強み
- シンプルな構造で処理しやすい
- ホームページ情報が豊富
- 掲載期間情報あり

### 弱み
- 業界分類・雇用形態・採用人数がない
- 代表者情報がない（セグメント分析不可）
- 電話番号が単一列のみ

### 補完戦略

他媒体との突合時に以下を補完：
1. PT・OT・STネットデータから雇用形態・リハビリ分類を取得
2. ミイダスデータから代表者情報・設立情報を取得
