# ハローワークリスト更新 マスタールール

> **最終確定日**: 2026-01-07
> **目的**: ハローワーク求人データをSalesforceと突合し、既存レコードの更新および新規リード候補の抽出を行う

---

## 🔴 絶対ルール（違反厳禁）

### 1. 成約先への架電事故防止
**成約先に営業架電してしまうと事故になる。絶対にやってはダメ。**

- 成約先は **電話番号 AND 法人番号** の両方で突合して除外
- 更新データ・新規リード候補の両方から除外必須

### 2. 取引開始済Leadの除外
**ConvertされたLeadはSalesforceで更新不可**

- `Status = '取引開始済'` のLeadは更新対象から除外

### 3. 本番更新前のバックアップ取得
**更新実行前に必ず全件バックアップを取得すること**

- Account / Contact / Lead の3オブジェクトすべてのバックアップを取得
- バックアップは更新対象の全レコード・全フィールドを含めること
- ファイル名: `backup_{Object}_{YYYYMMDD_HHMMSS}.csv`
- 保存先: `data/output/hellowork/`
- テスト用の少数件バックアップではなく、本番更新対象の全件を取得すること

```python
# バックアップ取得コマンド例
python -c "
from scripts.bulk_full_update import BulkFullUpdater
updater = BulkFullUpdater()
updater.authenticate()
# Bulk API 2.0 Queryで更新対象Idに基づいてバックアップ取得
"
```

## STEP 1: 電話番号正規化

### ルール
```python
def normalize_phone(val):
    """数字以外をすべて除去"""
    if pd.isna(val):
        return val
    digits = re.sub(r"[^\d]", "", str(val))
    return digits if digits else ''
```

### 対象カラム
- ハローワーク: `選考担当者ＴＥＬ` → `選考担当者ＴＥＬ_加工`
- Salesforce: 各オブジェクトの電話フィールド

### 🔴 取引開始済Leadの電話番号マッチング（重複防止）

**取引開始済Leadは「更新対象」からは除外するが、「電話番号マッチング」には必ず含めること。**

取引開始済Leadの電話番号をマッチングセットから除外すると、同じ電話番号で新規リードが重複作成される事故が発生する。

```python
# ❌ 間違い: 取引開始済を除外してからマッチング
lead = lead[lead['Status'] != '取引開始済']  # ← この電話番号がマッチングから漏れる
sf_phones.update(lead['Phone'].apply(normalize_phone))

# ✅ 正しい: 全Leadの電話番号でマッチング（取引開始済含む）
sf_phones.update(lead['Phone'].apply(normalize_phone))  # 全件の電話番号を含める
# 更新処理時のみ取引開始済を除外
update_target = matched[matched['Status'] != '取引開始済']
```

> **2026-03-02 障害発生**: セグメントA/B 331件中204件が取引開始済Leadの電話番号と重複。
> 原因: パイプラインが取引開始済Leadをマッチング対象から除外していた。
> 対処: 204件を削除、パイプライン修正済み。

### STEP 3-2: 職種キーワード除外

産業分類フィルタ通過後、さらに以下のキーワードを含む職種を除外する。

```python
# 除外キーワード（★ 2026-03-02更新: 保育士系は除外対象外に変更）
EXCLUDE_JOB_KW = ['幼稚園教諭', '歯科', '薬剤師', '訪問看護']

# ただし以下のキーワードを含む場合は除外しない（優先）
KEEP_JOB_KW = ['児童指導員', '放課後', '児童発達', '保育']

def should_exclude(job_title):
    if pd.isna(job_title):
        return False
    job = str(job_title)
    # 残すべきキーワードが含まれていたら除外しない
    for kw in KEEP_JOB_KW:
        if kw in job:
            return False
    # 除外キーワードに該当したら除外
    for kw in EXCLUDE_JOB_KW:
        if kw in job:
            return True
    return False

df = df[~df['職種'].apply(should_exclude)]
```

### 成約先の定義（Salesforceレポート条件）
```sql
Status__c LIKE '%商談中%'
OR Status__c LIKE '%プロジェクト進行中%'
OR Status__c LIKE '%深耕対象%'
OR Status__c LIKE '%過去客%'

OR RelatedAccountFlg__c = 'グループ過去案件実績あり'
```

### 突合キー（両方で突合 - 必須）

| キー | ハローワーク側 | 成約先側 |
|-----|---------------|---------|
| 電話番号 | 選考担当者ＴＥＬ_加工 | Phone |
| 法人番号 | 法人番号 | CorporateNumber__c, CorporateIdentificationNumber__c |

### 法人番号正規化
```python
def normalize_corp_num(num):
    """法人番号を正規化（10桁以上の数字のみ有効）"""
    if pd.isna(num) or not num:
        return ''
    num = str(num).strip()
    num = re.sub(r'[^0-9]', '', num)
    return num if len(num) >= 10 else ''
```

### 除外ロジック
```python
# 電話番号 OR 法人番号でマッチしたものを除外
phone_match = df['phone_norm'].isin(contract_phones)
corp_match = df['corp_norm'].isin(contract_corps)
is_contract = phone_match | corp_match

# 成約先を除外
filtered_df = df[~is_contract]
```

## ターゲットセグメント定義（2軸抽出方式）

> **目的**: ハローワークリストデータから、営業ターゲットとなる求人を抽出する
> **方式**: パターン①（職種分類）OR パターン②（産業分類）の2軸で抽出
> **適用タイミング**: セグメントごとに個別に実行可能（必ずしも同時実行ではない）
> **参照ドキュメント**:
> - `claudedocs/hellowork_job_classification.md`（職業分類 15大分類・99中分類・377小分類）
> - `claudedocs/hellowork_industry_classification.md`（産業分類 20大分類・99中分類）

---

### 🔴 2軸抽出の考え方

**産業分類 = 会社の「主たる事業」のみ**を示す。複数事業を営む会社の副業は反映されない。
**職種分類 = その求人の「仕事内容」**を示す。会社の業種に関係なく職種で判定できる。

**→ どちらか一方では漏れが生じる。両方を対等なOR条件で抽出することでMECEに近づける。**

```
【パターン②だけでは漏れるケース】
  不動産会社（K不動産業）が介護職員を募集
  → 産業分類: K不動産業 → 介護セグメントに該当しない → ❌ 漏れる
  → 職種分類: 050-01 高齢者入所型施設介護員 → 介護セグメント → ✅ パターン①で捕捉

  製造業（E製造業）が工場内建設工事の作業員を募集
  → 産業分類: E製造業 → 工事セグメントに該当しない → ❌ 漏れる
  → 職種分類: 092-01 建設・土木作業員 → 工事セグメント → ✅ パターン①で捕捉

【パターン①だけでは漏れるケース】
  介護施設（P医療・福祉）が一般事務員を募集
  → 職種分類: 034-01 一般事務員 → 介護セグメントに該当しない → ❌ 漏れる
  → 産業分類: P 85社会福祉・介護事業 → 介護セグメント → ✅ パターン②で捕捉

  建設会社（D建設業）が経理事務員を募集
  → 職種分類: 038-03 経理事務員 → 工事セグメントに該当しない → ❌ 漏れる
  → 産業分類: D建設業 06総合工事業 → 工事セグメント → ✅ パターン②で捕捉
```

| フィルタ | 何を見るか | カバー範囲 | 単独での弱点 |
|---------|-----------|-----------|------------|
| **パターン①（職種）** | 求人の仕事内容 | 産業に関係なく対象職種を抽出 | 事務・営業等の汎用職種が漏れる |

| **① OR ②** | **両方** | **相互補完でMECE** | — |

### 抽出ロジック（共通）

```python
def extract_segment(df, segment_config):
    """指定セグメントに該当する求人を抽出（パターン① OR ②）"""

    # パターン②の設定
    industry_codes = segment_config.get('industry_codes', [])
    industry_keywords = segment_config.get('industry_keywords', [])
    industry_whitelist = segment_config.get('industry_names_whitelist', [])  # 完全一致
    industry_exclude = segment_config.get('industry_names_exclude', [])     # 明示除外

    # パターン①の設定
    job_codes = segment_config.get('job_codes', [])
    job_major_prefixes = segment_config.get('job_major_prefixes', [])
    exclude_job_codes = segment_config.get('exclude_job_codes', [])
    exclude_job_kw = segment_config.get('exclude_job_keywords', [])
    keep_job_kw = segment_config.get('keep_job_keywords', [])

    def matches(row):
        ind_code = str(row.get('産業分類コード', '')).strip()
        ind_name = str(row.get('産業分類（名称）', '')).strip()
        job_code = str(row.get('職業分類コード', '')).strip()
        job_title = str(row.get('職種', ''))

        # --- 産業分類の明示除外チェック ---
        if ind_name in industry_exclude:
            return False

        # --- パターン②: 産業分類 ---
        # ホワイトリスト方式（セグメントA/B: 名称完全一致）
        if industry_whitelist and ind_name in industry_whitelist:
            return True
        # コード方式（セグメントC〜F）
        if ind_code in industry_codes:
            return True
        # キーワード方式（セグメントC〜F）
        for kw in industry_keywords:
            if kw in ind_name:
                return True

        # --- パターン①: 職種分類 ---
        # 除外対象の職種コードは先にスキップ
        if job_code in exclude_job_codes:
            return False

        # 職種キーワード除外（keep優先）
        if exclude_job_kw:
            has_keep = any(kw in job_title for kw in keep_job_kw)
            if not has_keep and any(kw in job_title for kw in exclude_job_kw):
                return False

        # 大分類プレフィックス一致
        for prefix in job_major_prefixes:
            if job_code.startswith(prefix):
                return True
        # 個別コード一致
        if job_code in job_codes:
            return True

        return False

    return df[df.apply(matches, axis=1)]
```

### 抽出フローの位置づけ（既存フローとの関係）

```
ハローワークCSV
    │
    ▼
[STEP 1] 電話番号正規化
    │
    ▼
[STEP 2] Salesforce突合（Account/Contact/Lead）
    │
    ├─→ マッチ → 既存更新フロー（変更なし）
    │
    └─→ 未マッチ
         │
         ▼
    [STEP 3] セグメント別ターゲット抽出 ← ★ 2軸方式に変更
         │  依頼に応じてセグメントA〜Fを個別/組み合わせで実行
         │  パターン①（職種） OR パターン②（産業）で抽出
         │
         ▼
    [STEP 4] 人口・従業員数フィルタ（任意）
         │
         ▼
    [STEP 5] 成約先除外（電話番号 + 法人番号）
         │
         ▼
    ★ セグメント別 new_leads_{segment}.csv
```

> **既存STEP 3（ホワイトリスト方式）からの変更点**:
> - 旧: `TARGET_INDUSTRIES` リスト（産業分類名称のみ）+ 職種キーワード除外

> - セグメントごとに個別実行可能で運用柔軟性が向上

```python
df['_pri'] = df['雇用形態'].map(EMPLOYMENT_PRIORITY).fillna(5)
```

### STEP 7: 決裁者近接スコア（★1〜5）

#### 判定指標

| 指標名 | 判定ロジック | 意味 |
|--------|------------|------|
| **代表者直通** | 代表者名 = 選考担当者名 | 電話すると代表が出る |
| **同姓親族** | 代表者と担当者の苗字が一致 | 親族経営の可能性 |
| **決裁者役職** | 担当者役職に決裁者キーワード含む | 権限者との接点あり |
| **管理部門壁** | 担当者部署が総務/人事/管理部 | 決裁者に到達困難 |
| **家族経営名** | 社名に代表者の苗字が含まれる | 同族企業の可能性 |
| **HP有無** | 事業所ホームページの有無 | 事前調査可否 |
| **携帯判定** | 電話番号が090/080/070始まり | 直接連絡の容易さ |

```python
# 決裁者キーワード
DECISION_MAKER_TITLES = [
    '代表取締役', '社長', '代表', '取締役', '役員',
    '理事長', '理事', '施設長', '院長', '園長',
    '所長', '会長', '専務', '常務', '部長',
    '支配人', '統括', 'オーナー', '経営者',
]

# 管理部門キーワード（壁）
MANAGER_WALL_KEYWORDS = ['総務', '人事', '管理部', '管理課', '事務局', '労務']
```

#### スコア判定ロジック

| ランク | スコア | 条件 | 営業的意味 |
|--------|-------|------|-----------|
| ★★★★★ | 5 | 代表者直通 + 携帯電話 | 最強。代表に直接携帯で繋がる |
| ★★★★☆ | 4 | 代表者直通 / 同姓親族+小規模 / 決裁者役職+携帯 | 決裁者に高確率で到達 |
| ★★★☆☆ | 3 | 同姓親族 / 決裁者役職 / 家族経営名+小規模 | 決裁者接点あり |
| ★★☆☆☆ | 2 | 上記いずれにも該当しない | 一般担当者 |
| ★☆☆☆☆ | 1 | 管理部門壁あり | 総務/人事がブロック |

```python
def calc_proximity_score(row):
    is_direct = row['代表者直通'] == '○'
    is_mobile = row['電話番号_携帯判定'] == '携帯'
    is_family = row['同姓親族'] == '○'
    is_dm_title = row['決裁者役職'] == '○'
    is_wall = row['管理部門壁'] == '○'
    is_small = row['従業員数_数値'] <= 10
    is_family_biz = row['家族経営名'] == '○'

    if is_direct and is_mobile:             return 5  # ★★★★★
    if is_direct:                           return 4  # ★★★★☆
    if is_family and is_small:              return 4
    if is_dm_title and is_mobile:           return 4
    if is_family:                           return 3  # ★★★☆☆
    if is_dm_title:                         return 3
    if is_family_biz and is_small:          return 3
    if is_wall:                             return 1  # ★☆☆☆☆
    return 2                                           # ★★☆☆☆
```

```python
def clean_corporate_number(val):
    s = str(val).strip()
    if '.' in s:
        s = str(int(float(s)))  # .0除去
    s = re.sub(r'[^\d]', '', s)
    return s if len(s) == 13 else ''

# 日付フォーマット
def clean_date(val):
    s = str(val).strip()[:10]
    s = s.replace('/', '-')  # YYYY/MM/DD → YYYY-MM-DD
    if re.match(r'^\d{4}-\d{2}-\d{2}$', s):
        return s
    return ''

# メールバリデーション
def validate_email(email):
    if not re.match(r'^[^@]+@[^@]+\.[^@]+$', email):
        return ''  # TLDなしは無効
    return email
```
