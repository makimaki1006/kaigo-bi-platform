## データソース仕様

### ファイル形式
- **ファイル名パターン**: `miidas_full_results_YYYYMMDD_HHMM_with_contacts.csv` または `miidas_structured_data.csv`
- **形式**: CSV
- **エンコーディング**: UTF-8 または UTF-8-sig（ファイルにより異なる、読み込み時に確認すること）
- **列数**: 197〜278列（データソースにより変動、多数のコンピテンシー関連列を含む）

| url | ミイダス求人ページURL | Yes | 全件 |
| 企業名 | 会社名 | Yes | 3050件 |
| 連絡先 | 電話番号・メール・担当者名を含むテキスト | Yes | 2037件 |
| 勤務地 | 勤務地住所 | No | 2661件 |
| 本社住所 | 本社所在地 | No | 3050件 |

| 企業規模 | 従業員規模 | No | 3050件 |
| 設立 | 設立年代 | No | 3050件 |

### 連絡先フィールドの特徴

## 電話番号抽出ロジック

### 正規表現パターン

```python
patterns = [
    r'0\d{1,4}[-－ー\s]?\d{1,4}[-－ー\s]?\d{3,4}',  # ハイフン区切り
    r'0\d{9,10}',  # 連続数字
]
```

### 携帯電話判定

```python
def is_mobile(phone: str) -> bool:
    """070/080/090で始まる場合は携帯電話"""
    normalized = re.sub(r'[^0-9]', '', phone)
    return normalized.startswith(('070', '080', '090'))
```

## 職種抽出ロジック

`全本文ダンプ` の最初の行から職種を抽出：

```
《1週間〜10日の連休あり》歯科衛生士【実務未経験の方も歓迎...
↓
歯科衛生士
```

### 抽出パターン

```python
def extract_job_title(text: str) -> str:
    """全本文ダンプから職種を抽出"""
    if not text:
        return ""
    first_line = text.split('\n')[0]
    # 《》【】などの装飾を除去
    cleaned = re.sub(r'[《》【】〈〉「」『』]', '', first_line)
    return cleaned.strip()[:100]  # 最大100文字
```

## セグメント分析（ミイダス固有）

### セグメント定義

ミイダスは「代表者」フィールドを持つため、セグメント分析が可能。

| セグメント | 定義 | 優先度 |
|-----------|------|--------|
| **代表者直通** | 担当者名 = 代表者名（部分一致含む） | ★★★ 最強 |
| **担当者経由** | 担当者名 ≠ 代表者名 | ★★ |
| **代表者情報なし** | 代表者フィールドが空 | ★ |
| **バイネームなし** | 担当者名が「担当者」または空 | - |

### 電話番号タイプ

| タイプ | 判定 |
|-------|------|
| 携帯あり | MobilePhoneに値あり（携帯のみ + 両方） |
| 携帯なし | MobilePhoneが空 |

### 最強セグメント

**「代表者直通 × 携帯あり」** が最も成約率の高いセグメント。

→ **市来に優先割当**（MASTER_RULE_paid_media.md §所有者割り当てルール参照）

### 2026-01-08 分析結果

| セグメント | 携帯あり | 携帯なし | 合計 | 携帯率 |
|-----------|---------|---------|------|--------|
| 代表者直通 | 32 | 21 | 53 | 60.4% |
| 担当者経由 | 30 | 31 | 61 | 49.2% |
| 代表者情報なし | 0 | 0 | 0 | - |
| バイネームなし | 147 | 232 | 379 | 38.8% |
| **合計** | **209** | **284** | **493** | **42.4%** |

**最強セグメント: 32件（6.5%）**

## 処理フロー

```
1. データ読み込み
   ├─ ミイダスデータ（CSV）
   ├─ 成約先電話番号（CSV）
   ├─ 電話済み電話番号（Excel全シート）
   └─ Salesforce既存データ（Lead/Account/Contact）

2. ★ 職種フィルタ（保育/非保育 分離）
   ├─ 全本文ダンプ + 企業名からキーワード判定
   ├─ 対象職種 → 処理続行
   └─ 非対象職種 → 別管理（所有者・レポートを分ける）
   ※ 詳細は下記「職種フィルタ」セクション参照

3. 電話番号抽出
   └─ 連絡先フィールドから正規表現で電話番号を抽出

4. 担当者名・役職抽出
   ├─ 連絡先フィールドから担当者名を抽出
   ├─ 役職プレフィックスを分離 → Title
   └─ 代表者フィールドをPresidentName__cにマッピング

5. 除外処理
   ├─ 成約先電話番号一致 → 除外（理由: 成約先電話番号）
   └─ 電話済み電話番号一致 → 除外（理由: 電話済み）

6. 突合処理
   ├─ 電話番号で既存レコード検索
   ├─ マッチあり → 更新対象
   └─ マッチなし → 新規リード候補

7. セグメント分析
   ├─ 代表者直通判定
   ├─ 携帯電話有無判定
   └─ 最強セグメント特定

8. CSV生成（対象/非対象を分離して出力）
   ├─ miidas_new_leads_*.csv（新規リード作成用）
   ├─ miidas_lead_updates_*.csv（Lead更新用）
   ├─ miidas_account_updates_*.csv（Account更新用）
   └─ miidas_excluded_*.csv（除外リスト）

9. 所有者割り当て（対象/非対象で異なる場合あり）
   └─ MASTER_RULE_paid_media.md §所有者割り当てルール参照

10. Salesforceインポート（ユーザー確認後）

11. メモ欄差別化
    └─ MASTER_RULE_paid_media.md §メモ欄差別化参照
```

## 🔴 職種フィルタ（対象/非対象の分離）

### なぜ必要か

ミイダスのテーマ別リスト（例: 「保育園リスト」）であっても、**非対象職種のレコードが混入する**。
2026-03-05の保育園リスト（339件）では34件（10%）が非保育（警備、葬祭、建設、不動産、事務等）だった。

**混入を放置すると、対象外の企業に架電してしまう事故が起きる。必ずフィルタすること。**

### フィルタロジック

```python
# 保育園リストの場合のキーワード例
HOIKU_KEYWORDS = ['保育', '幼稚園', '認定こども園', '託児']

def is_target_record(row, keywords):
    """全本文ダンプ + 企業名にキーワードが含まれるか判定"""
    text = str(row.get('全本文ダンプ', '')) + str(row.get('企業名', ''))
    return any(k in text for k in keywords)
```

### 分離後の処理

| 分類 | 所有者（2026-03-05実績） | 処理 |
|------|----------------------|------|
| **対象（保育）** | 深堀 | 通常の新規作成/更新 |
| **非対象（その他）** | 小林 | 別途所有者割当、メモで区別 |

### 2026-03-05 保育園リスト分離結果

| 分類 | 元データ | 新規リード | Lead更新 | Account更新 |
|------|---------|-----------|---------|------------|
| 保育 | 305件 | 57件 | 69件 | 33件 |
| 非保育 | 34件 | 9件 | 2件 | 0件 |
| 判定不能 | - | - | 14件 | 6件 |

## 🔴 既知の問題と対処法

### Salesforceフィールドの制約

| 問題 | 原因 | 対処 |
|-----|------|------|
| Lead.Description が存在しない | 本orgのLead上に標準Descriptionフィールドがない | Description更新をスキップ |
| LastName にスペースを含むとエラー | カスタムバリデーションルール（記号・スペース禁止） | LastNameを除外して他フィールドのみ更新 |
| Paid_NumberOfRecruitment__c 型エラー | xsd:double 型に「3名」等のテキストを入れようとする | 数値部分のみ抽出（例: 「3名」→「3」） |
| Paid_JobTitle__c / Paid_RecruitmentType__c 長すぎ | 文字列長制限超過 | 100文字に截断 |
| CANNOT_UPDATE_CONVERTED_LEAD | コンバート済みリードは更新不可 | スキップ（対処不要） |

### Account追跡の制約

AccountにはLeadSourceMemo__cがないため、メモ欄で「ミイダス更新分」を追跡できない。

**代替手段**: `Paid_DataExportDate__c` + `Paid_DataSource__c = 'ミイダス'` でレポートフィルタ
