# FS引継ぎメモ品質改善プロジェクト

**ステータス**: Phase 1 完了（Gemプロンプト v2 作成済み、テスト待ち）

---

## プロジェクト概要

**目的**: Project__c の FS引継ぎメモ6項目の入力品質を向上させ、役務（カスタマーサクセス）への引継ぎ精度を高める

**背景**: Salesforce上のProject__cオブジェクトには、FS（フィールドセールス）が商談後に記入する引継ぎメモ6項目がある。現状は記入率・記入品質にばらつきがあり、役務が顧客対応を始める際に情報不足が発生している。

**対象6項目**:
1. 募集職種
2. 採用課題
3. 導入ポイント（契約の決め手）
4. 刺さったポイント（商談中に響いた提案）
5. 先方関係者情報
6. 担当者注意点

## プロジェクト経緯

### Phase 1: 現状分析（2026-01-29）

**実施内容**: Project__cの既存データを分析し、引継ぎメモの品質課題を特定

- Salesforce上の実データを確認し、記入率・記入品質を調査
- 6項目それぞれの「良い例」「悪い例」を抽出
- 品質課題を以下の4カテゴリに整理:
  - 記入率のばらつき（項目によって空欄が多い）
  - 記入内容の抽象度（「いい人」等の曖昧な表現）
  - フォーマットの不統一
  - 必要情報の欠落

**成果物**:
- `01_analysis/GOOD_EXAMPLES.md` - 実データからの良い記入例集
- `01_analysis/FIELD_DEFINITIONS.md` - 各項目の定義・意図・記入基準

### Phase 2: 改善計画策定（2026-01-29）

**実施内容**: 品質改善のアプローチを4本柱で設計

4つの改善アプローチ:
1. **フォーム改善** - Salesforce入力画面の改善（ガイドテキスト、プレースホルダー）
2. **チェックリスト** - 商談前後のチェックリスト導入
3. **テンプレート** - 記入テンプレートの整備
4. **モニタリング** - 品質スコアリングと定期レビュー

**成果物**:
- `02_design/IMPROVEMENT_PLAN.md` - 4本柱の改善計画

### Phase 3: AI自動生成システム設計（2026-01-29）

**実施内容**: 商談の文字起こしから引継ぎメモを自動生成するシステムの要件定義

**方針決定の経緯**:
- 当初はUIを含む独自システムを検討
- ユーザー判断: UIは不要。Gemini Gem（カスタムChatBot）を使用する
- 商談文字起こし・電話音声をGemに投入して6項目を自動生成する運用

**Zoom Phone連携の構想**:
- Salesforce Project__c → 取引先責任者の電話番号取得
- Zoom Phone API → 通話ログ検索 → 録音ID → 文字起こしダウンロード
- 現時点は手動運用、将来的にAPI自動化

**成果物**:
- `02_design/AI_GENERATOR_PRD.md` - 要件定義書（Zoom Phone API設計含む）

### Phase 4: Gem プロンプト作成・改善（2026-01-29）

**実施内容**: Gemini Gem用のプロンプトを作成し、複数回のレビューで精度を向上

**改善の経緯**:

| 改善ポイント | 変更前 | 変更後 |
|-------------|--------|--------|
| 業務背景 | 「人材紹介会社」と記載 | 「採用コンサルティングサービス」に修正 |
| 出力フォーマット | ネストしたコードブロック（Gem非対応） | `=====`区切りに変更 |
| 抽出力 | 基本的な抽出指示のみ | 「こういう発言を見逃さない」リスト追加 |
| 不足情報対応 | 「※情報不足」で終了 | 3段階対応（FS確認→求人票回収→役務申し送り） |
| 求人票との切り分け | 全情報をヒアリングで取得する前提 | 求人票で分かる情報と会話でしか得られない情報を明確に分離 |
| 確認事項の網羅性 | 2項目のみ | 4カテゴリ9項目に拡充 |

**「求人票では分からない情報」の整理**:

求人票に載る情報（無理にヒアリング不要）:
- 給与、勤務時間、休日、資格要件、勤務地、雇用形態

会話からしか得られない情報（確認必須）:
- 面接の重視点、合わなかった人の特徴、書類選考基準、定着vs即戦力
- 連絡手段の好み、繋がりやすい/避けるべき時間帯
- 現場と経営の温度差
- 過去の採用活動への不満、早期退職の理由

**成果物**:
- `03_gem_prompt/GEM_PROMPT_FINAL.md` - ★最終版プロンプト（v2）

## 運用フロー

```
Zoom商談（録画）
    │ 文字起こし
    ↓
Gemini Gem に貼り付け
    │
    │ ← 電話メモ/Zoom Phone文字起こしがあれば追加投入
    │
    ↓
6項目の引継ぎメモ自動生成
    │
    │ ← 不足情報の確認（FSメモリ→求人票→先方確認→役務申し送り）
    │
    ↓
Salesforce Project__c に貼り付け
```
