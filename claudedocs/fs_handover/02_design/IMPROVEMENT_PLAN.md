# FS引継ぎメモ 品質改善プラン

## 概要

### 現状の問題点
| 問題 | 影響 |
|------|------|
| 内容が薄い（26-37%が50文字以下） | 役務が顧客理解できず、対応品質低下 |
| テンプレートヘッダーのみで中身なし | 情報として機能しない |
| 「特になし」「ー」で済ませる | 成約理由・注意点が不明 |
| FSが成約理由を理解していない | 再現性のある営業ができない |

### 改善の4本柱
1. **入力フォームの改善** - 必須項目化・ガイド質問
2. **チェックリスト作成** - 引継ぎ前の確認項目
3. **良い例のテンプレート化** - 具体例による品質基準明示
4. **品質モニタリング** - 自動検知・アラート

## 1. 入力フォームの改善

### 1.1 必須項目の設定

| フィールド | 現状 | 提案 | 理由 |
|-----------|------|------|------|
| 募集職種 | 任意 | **必須** | 求人の基本情報 |
| 採用課題 | 任意 | **必須** | 提案の起点となる情報 |
| 導入ポイント | 任意 | **必須** | 成約理由の核心 |
| 刺さったポイント | 任意 | **必須** | 役務の対応に直結 |
| 先方関係者情報 | 任意 | **必須** | コミュニケーションの基盤 |
| 担当者注意点 | 任意 | 推奨 | あれば有益 |

### 1.2 ガイド質問の追加

各フィールドに以下のガイド質問を表示：

#### 募集職種
```
【回答すべき質問】
□ 募集職種名は？（例：介護職、看護師、PT等）
□ 雇用形態は？（正社員/パート/派遣）
□ 勤務地・配属先は？
□ 必須資格・経験は？
□ 給与・待遇の特徴は？
□ シフト・勤務時間の特徴は？
```

#### 採用課題
```
【回答すべき質問】
□ なぜ今回採用が必要になった？（退職/増員/新規開設）
□ これまでどんな採用活動をした？（媒体/紹介会社）
□ うまくいかなかった理由は？
□ 急ぎ度合いは？（いつまでに何名必要）
□ 採用の決裁者は誰？
```

#### 導入ポイント
```
【回答すべき質問】
□ 他社ではなく当社を選んだ決め手は？
□ 価格面での評価は？
□ サービス内容で評価された点は？
□ 営業担当への評価は？
□ 契約時に懸念されていた点とその解消方法は？
```

#### 刺さったポイント
```
【回答すべき質問】
□ 商談中、先方が最も反応した提案・情報は？
□ 「それいいね」と言われた具体的なポイントは？
□ 競合と比較して評価された点は？
□ 先方の課題に対してどう解決策を提示した？
```

#### 先方関係者情報
```
【回答すべき質問】
□ 採用担当者の氏名・役職・連絡先は？
□ 決裁者は誰？（担当者と同じ/別の場合は誰）
□ 現場責任者は誰？（面接官になる人）
□ それぞれの人柄・コミュニケーションスタイルは？
□ 連絡手段の好み（電話/メール/LINE）は？
```

#### 担当者注意点
```
【回答すべき質問】
□ 連絡のタイミングで避けるべき時間帯は？
□ NGワード・触れてはいけない話題は？
□ 先方が重視する価値観・こだわりは？
□ 過去にトラブルになった事例は？
□ 特別な対応が必要な事項は？
```

### 1.3 最低文字数の設定

| フィールド | 最低文字数 | 理由 |
|-----------|-----------|------|
| 募集職種 | 100文字 | 最低限の求人情報 |
| 採用課題 | 80文字 | 背景説明に必要 |
| 導入ポイント | 100文字 | 成約理由の説明 |
| 刺さったポイント | 80文字 | 具体的なポイント |
| 先方関係者情報 | 150文字 | 人物情報には詳細が必要 |
| 担当者注意点 | 50文字 | 短くても可 |

## 2. チェックリスト作成

### 2.1 引継ぎ前チェックリスト（FS用）

```
═══════════════════════════════════════════════════
FS引継ぎ前 自己チェックリスト
═══════════════════════════════════════════════════

【基本情報】
□ 募集職種・雇用形態・勤務地を明記した
□ 必須資格・経験要件を明記した
□ 給与・待遇の特徴を記載した

【採用背景】
□ なぜ採用が必要になったか説明した
□ 過去の採用活動と課題を記載した
□ 採用の緊急度・期限を明記した

【成約理由】
□ 当社を選んだ決め手を具体的に書いた
□ 先方が評価したポイントを記載した
□ 商談中に「刺さった」提案を記録した

【関係者情報】
□ 担当者の氏名・役職・連絡先を記載した
□ 決裁者が誰か明記した
□ 人柄・コミュニケーションスタイルを記載した

【注意事項】
□ 連絡のタイミング・手段の好みを記載した
□ NGワード・避けるべき話題を記載した
□ 特別な対応事項を記載した

【品質チェック】
□ 各項目が50文字以上ある
□ 「特になし」「ー」で済ませていない
□ 役務が読んで顧客対応できる内容になっている
═══════════════════════════════════════════════════
```

### 2.2 引継ぎ完了チェックリスト（役務用）

```
═══════════════════════════════════════════════════
役務引継ぎ受領 確認チェックリスト
═══════════════════════════════════════════════════

【確認項目】
□ 募集職種と求める人材像が理解できた
□ 採用の背景・緊急度が理解できた
□ 成約の決め手が理解できた
□ 担当者・決裁者が誰か把握できた
□ 連絡方法・タイミングの注意点が分かった

【不足情報の確認】
□ 不明点をFSに質問した
□ FSから追加情報をもらった

【引継ぎ品質評価】
この引継ぎ内容で顧客対応できますか？
○ 十分対応できる
○ 対応できるが一部不明点あり → FSに確認
○ 対応困難、情報不足 → FSに再作成依頼
═══════════════════════════════════════════════════
```

## 3. 良い例のテンプレート化

### 3.1 募集職種テンプレート

**良い例（社会福祉法人清香会）を参考：**

```
【職種】
・募集職種：[職種名]
・雇用形態：[正社員/パート/派遣]
・勤務地：[施設名・住所]

【求める人材】
・必須資格：[資格名]
・必須経験：[経験年数・内容]
・歓迎条件：[あれば記載]

【勤務条件】
・給与：[金額・手当]
・勤務時間：[シフト詳細]
・休日：[休日数・曜日]

【特記事項】
・[その他アピールポイント]
```

### 3.2 採用課題テンプレート

**良い例（社会福祉法人ざおう福祉会）を参考：**

```
【採用の背景】
・理由：[退職補充/増員/新規開設]
・詳細：[具体的な経緯]

【これまでの採用活動】
・利用媒体/紹介会社：[具体名]
・結果：[応募数・採用数・課題]

【採用の課題】
・課題①：[具体的な課題]
・課題②：[具体的な課題]

【採用要件】
・人数：[必要人数]
・期限：[いつまでに]
・緊急度：[高/中/低]

【決裁フロー】
・決裁者：[役職・氏名]
・決裁までの流れ：[プロセス]
```

### 3.3 導入ポイントテンプレート

**良い例（株式会社イデアル）を参考：**

```
【成約の決め手】
Q: なぜ当社を選んだ？
A: [先方の回答をそのまま記載]

【評価されたポイント】
・価格面：[評価内容]
・サービス内容：[評価内容]
・営業対応：[評価内容]

【競合との比較】
・競合他社：[社名]
・当社が勝った理由：[具体的な差別化ポイント]

【契約時の懸念と解消】
・懸念点：[何を心配していたか]
・解消方法：[どう説明して解消したか]
```

### 3.4 刺さったポイントテンプレート

```
【商談で響いたポイント】
・提案①：[具体的な提案内容]
  → 反応：[先方の反応]

・提案②：[具体的な提案内容]
  → 反応：[先方の反応]

【先方の課題と解決策】
・課題：[先方が抱えていた課題]
・提示した解決策：[当社の提案]
・先方の評価：[どう受け止められたか]
```

### 3.5 先方関係者情報テンプレート

**良い例（株式会社イデアル）を参考：**

```
【採用担当者】
・氏名：[フルネーム]
・役職：[役職名]
・連絡先：[電話/メール]
・人柄：[性格・コミュニケーションスタイル]
・連絡の好み：[電話/メール/LINE、時間帯]

【決裁者】（担当者と異なる場合）
・氏名：[フルネーム]
・役職：[役職名]
・関与度：[どの程度関わるか]

【現場責任者】（面接官になる方）
・氏名：[フルネーム]
・役職：[役職名]
・面接での重視点：[何を見るか]

【その他キーパーソン】
・[必要に応じて追加]
```

### 3.6 担当者注意点テンプレート

**良い例（社会福祉法人ざおう福祉会）を参考：**

```
【連絡について】
・好ましい連絡手段：[電話/メール/LINE]
・避けるべき時間帯：[具体的な時間]
・レスポンスの特徴：[返信早い/遅いなど]

【コミュニケーションスタイル】
・話し方の好み：[結論から/丁寧に説明など]
・避けるべき話題：[NGトピック]

【価値観・こだわり】
・重視していること：[具体的に]
・嫌がること：[具体的に]

【過去のトラブル・注意事項】
・[過去の問題があれば記載]

【特別対応】
・[必要な配慮事項]
```

## 4. 品質モニタリング

### 4.1 自動検知ルール

| 検知項目 | 条件 | アラートレベル |
|---------|------|---------------|
| 文字数不足 | 必須項目が50文字未満 | 🔴 高 |
| 空欄 | 必須項目が空 | 🔴 高 |
| プレースホルダー | 「特になし」「ー」「未定」のみ | 🟡 中 |
| テンプレートのみ | ヘッダーあり・内容なし | 🟡 中 |
| 古い情報 | 30日以上更新なし | 🟢 低 |

### 4.2 品質スコア計算

```
品質スコア =
  (募集職種スコア × 0.15) +
  (採用課題スコア × 0.15) +
  (導入ポイントスコア × 0.20) +
  (刺さったポイントスコア × 0.20) +
  (先方関係者情報スコア × 0.20) +
  (担当者注意点スコア × 0.10)

各項目スコア:
  100点: 充実（150文字以上かつガイド質問網羅）
   70点: 標準（80-149文字）
   40点: 最低限（50-79文字）
    0点: 不足（50文字未満または空欄）
```

### 4.3 モニタリングレポート

**週次レポート項目：**
- FS別 平均品質スコア
- 品質スコア分布（優良/標準/要改善）
- 最も改善が必要な項目
- 前週比較

**月次レポート項目：**
- FS別 品質推移グラフ
- ベストプラクティス事例
- 改善トレンド分析

### 4.4 フィードバックフロー

```
品質スコア70点未満
    ↓
自動アラート（FS本人＋マネージャー）
    ↓
24時間以内に修正
    ↓
役務確認・承認
    ↓
引継ぎ完了
```

| 施策 | 工数 | 効果 | 優先度 |
|------|------|------|--------|
| ガイド質問の追加 | 低 | 高 | ★★★ |
| チェックリスト配布 | 低 | 中 | ★★★ |
| テンプレート共有 | 低 | 高 | ★★★ |
| 最低文字数の設定 | 中 | 高 | ★★☆ |
| 必須項目化 | 中 | 高 | ★★☆ |
| 品質スコア自動計算 | 高 | 高 | ★☆☆ |
| 自動アラート | 高 | 中 | ★☆☆ |

### 推奨実装順序

1. **即座に実施可能**（今週）
   - チェックリストの配布
   - テンプレートの共有
   - ガイド質問のドキュメント化

2. **短期**（1-2週間）
   - Salesforceフォームへのガイド質問追加
   - バリデーションルール（最低文字数）設定

3. **中期**（1ヶ月）
   - 品質スコア計算フローの構築
   - 週次モニタリングレポート作成
