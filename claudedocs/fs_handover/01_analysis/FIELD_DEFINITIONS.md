# FS引継ぎメモ 入力項目定義書

## 本ドキュメントの目的

FS引継ぎメモの各項目について、**何を書くべきか**（定義）と**なぜ書く必要があるのか**（意図）を明確にし、質の高い引継ぎを実現する。

---

## 項目一覧

| No | 項目名 | 必須度 | 主な利用者 |
|----|--------|--------|-----------|
| 1 | 募集職種 | ★★★ | 役務・CA |
| 2 | 採用課題 | ★★★ | 役務・CA |
| 3 | 導入ポイント | ★★★ | 役務・FS（振り返り） |
| 4 | 刺さったポイント | ★★★ | 役務・FS（ナレッジ共有） |
| 5 | 先方関係者情報 | ★★★ | 役務 |
| 6 | 担当者注意点 | ★★☆ | 役務 |

## 1. 募集職種

### 定義
**クライアントが採用したい人材の職種・条件・待遇を記載する項目**

### 意図・目的

| 観点 | 説明 |
|------|------|
| **誰のため** | 役務・CA（キャリアアドバイザー） |
| **なぜ必要** | 求職者とのマッチング精度を高めるため |
| **どう使われる** | 求職者への求人紹介時の説明材料、マッチング判断の基準 |

### この項目がないと何が起きるか
- 役務が求人内容を正確に把握できず、ミスマッチな候補者を紹介してしまう
- CAが求職者に魅力的な説明ができず、応募意欲を引き出せない
- 面接後に「思っていた条件と違った」と辞退が発生する

### 記載すべき内容

| 項目 | 説明 | 例 |
|------|------|-----|
| 職種名 | 募集している職種 | 介護職、看護師、PT、事務 |
| 雇用形態 | 正社員/パート/派遣など | 正社員、パート（週3日〜） |
| 勤務地 | 施設名・住所 | ○○デイサービス（東京都中野区） |
| 必須資格 | 必ず必要な資格 | 介護福祉士、普通自動車免許 |
| 必須経験 | 必要な経験年数・内容 | 介護経験3年以上 |
| 歓迎条件 | あれば望ましい条件 | 認知症ケア経験者歓迎 |
| 給与 | 月給/時給、手当含む | 月給22万円〜（夜勤手当別途） |
| 勤務時間 | シフト、残業有無 | 日勤のみ、残業月10時間程度 |
| 休日 | 年間休日、曜日 | 年間休日110日、シフト制 |
| 特記事項 | その他アピール点 | 車通勤可、託児所あり |

### 良い例
```
【正職員】介護職 3名
・勤務地：○○特養（東京都練馬区）
・必須：介護福祉士
・歓迎：特養経験者
・月給：23万円〜28万円（夜勤4回込み）
・夜勤：月4回程度（夜勤なし相談可）
・休日：年間110日、シフト制
・特徴：残業少なめ、有給取得率80%以上
```

### NG例
```
介護職募集
```
→ これだけでは何も分からない

## 2. 採用課題

### 定義
**クライアントがなぜ採用活動をしているのか、どんな困りごとを抱えているかを記載する項目**

### 意図・目的

| 観点 | 説明 |
|------|------|
| **誰のため** | 役務・CA |
| **なぜ必要** | クライアントの本当のニーズを理解し、適切な対応をするため |
| **どう使われる** | 候補者選定の優先度判断、フォローアップの頻度・内容決定 |

### この項目がないと何が起きるか
- 役務が緊急度を把握できず、優先順位を誤る
- クライアントの本当の課題に応えられない候補者を紹介してしまう
- 「なぜうちに頼んだのか」が分からず、関係構築が表面的になる

### 記載すべき内容

| 項目 | 説明 | 例 |
|------|------|-----|
| 採用の背景 | なぜ今採用が必要か | 退職補充、増員、新施設オープン |
| 具体的な経緯 | 背景の詳細 | 3月末で2名退職予定、4月オープン |
| 過去の採用活動 | これまで何をしたか | ハローワーク、Indeed、他紹介会社 |
| うまくいかなかった理由 | なぜ成功していないか | 応募少ない、辞退多い、マッチしない |
| 採用の緊急度 | いつまでに何名必要か | 3月末までに2名必須 |
| 決裁者 | 採用を決める人 | 施設長、人事部長 |
| 決裁フロー | 採用決定までの流れ | 人事面接→施設長面接→内定 |

### 良い例
```
【背景】
2024年3月末で介護職2名が退職予定（結婚・転居）。
現状ギリギリのシフトで回しており、補充必須。

【これまでの活動】
・ハローワーク：6ヶ月掲載、応募3件、採用0
・Indeed：月5万円で3ヶ月、応募10件、面接5件、採用0（辞退）
・理由：給与が地域平均より低め、知名度がない

【緊急度】
4月1日までに最低1名、理想は2名。
1名も採用できないとシフトが回らない。

【決裁】
施設長が最終決定。面接は施設長＋介護主任の2名体制。
```

### NG例
```
人手不足のため
```
→ どこも人手不足。具体的な状況が分からない

## 3. 導入ポイント

### 定義
**クライアントが当社と契約を決めた理由・決め手を記載する項目**

### 意図・目的

| 観点 | 説明 |
|------|------|
| **誰のため** | 役務・FS自身（振り返り）・組織（ナレッジ） |
| **なぜ必要** | 役務がクライアントの期待値を把握するため、FSが成約要因を言語化するため |
| **どう使われる** | 役務の対応方針決定、FS営業スキル向上、組織の勝ちパターン蓄積 |

### この項目がないと何が起きるか
- 役務がクライアントの期待を把握できず、期待外れの対応をしてしまう
- FSが「なぜ成約できたか」を理解せず、再現性のある営業ができない
- 組織として勝ちパターンが蓄積されない

### 記載すべき内容

| 項目 | 説明 | 例 |
|------|------|-----|
| 選んだ理由（先方の言葉） | 契約の決め手を先方の言葉で | 「価格が安かったから」「対応が早かったから」 |
| 価格面の評価 | 価格についてどう思われたか | 他社より15%安い、予算内だった |
| サービス面の評価 | サービス内容の評価 | 求人票改善の提案が良かった |
| 営業対応の評価 | FS自身への評価 | レスポンスが早い、丁寧だった |
| 競合との比較 | 他社と比べてどうだったか | A社より安い、B社より対応が早い |
| 契約時の懸念 | 契約前に心配していたこと | 本当に採用できるか不安だった |
| 懸念の解消方法 | どう解消したか | 同業他社の成功事例を提示 |

### 良い例
```
【契約の決め手】（先方の言葉）
「他社3社から見積もりを取ったが、御社が一番安かった。
 あと、営業の○○さんの対応が早くて丁寧だったのが決め手」

【評価ポイント】
・価格：他社平均より約15%安い
・対応：問い合わせから見積もり提出まで2日
・提案：うちの施設に合った求人票の書き方を提案してくれた

【競合比較】
・A社：価格高い（当社の1.2倍）
・B社：対応遅い（見積もりに1週間）
・C社：提案なし、紹介するだけ

【懸念と解消】
・懸念：「過去に紹介会社で失敗している。本当に採用できるか」
・解消：同地域・同規模の成功事例を3件提示。初月保証も説明。
```

### NG例
```
価格とサービスが良かったから
```
→ 具体性がない。何が良かったのか分からない

## 4. 刺さったポイント

### 定義
**商談の中でクライアントが特に反応した提案・情報を記載する項目**

### 意図・目的

| 観点 | 説明 |
|------|------|
| **誰のため** | 役務・FS自身・組織（ナレッジ共有） |
| **なぜ必要** | 役務がクライアントの関心事を把握するため、FSの提案力向上のため |
| **どう使われる** | 役務のコミュニケーション、FS営業研修、成功事例の横展開 |

### この項目がないと何が起きるか
- 役務がクライアントの「ツボ」を把握できず、的外れな対応をする
- FSが何が効果的だったか振り返れず、スキルが向上しない
- 組織として有効な提案パターンが共有されない

### 「導入ポイント」との違い

| 項目 | 導入ポイント | 刺さったポイント |
|------|-------------|-----------------|
| タイミング | 契約決定時 | 商談中 |
| 内容 | 最終的な決め手 | 途中で響いた提案 |
| 視点 | 結果（Why） | プロセス（How） |

### 記載すべき内容

| 項目 | 説明 | 例 |
|------|------|-----|
| 響いた提案 | 先方が反応した提案内容 | 求人票改善の提案、成功事例の紹介 |
| 先方の反応 | どんな反応だったか | 「それいいね」「そういう視点はなかった」 |
| 課題と解決策 | 先方の課題に対する提案 | 応募が来ない→ターゲット絞り込み提案 |
| 競合との差別化 | 他社と違った点 | 紹介するだけでなく改善提案もする |

### 良い例
```
【響いた提案①】求人票改善
・提案：「御社の求人票、もったいないです」と言って改善点を3つ指摘
  - タイトルが一般的すぎる
  - 給与の書き方が分かりにくい
  - 写真がない
・反応：「そういう視点はなかった。具体的でありがたい」

【響いた提案②】成功事例
・提案：同地域・同規模の施設で3名採用できた事例を紹介
・反応：「具体的でイメージしやすい。うちでもできそう」

【響いた提案③】リスクフリー
・提案：成功報酬型なので初期費用ゼロ、採用できなければ費用なし
・反応：「試しやすい。これが決め手になった」

【課題への解決策提示】
・課題：「応募が来ない」
・提案：ターゲット層（40代女性、ブランク明け）に絞った訴求
・評価：「今までの紹介会社は紹介するだけだった。提案してくれるのは初めて」
```

### NG例
```
特になし
```
→ 成約したのに刺さったポイントがないはずがない

## 5. 先方関係者情報

### 定義
**クライアント側の関係者（担当者・決裁者・現場責任者など）の情報を記載する項目**

### 意図・目的

| 観点 | 説明 |
|------|------|
| **誰のため** | 役務（最重要） |
| **なぜ必要** | 役務が適切な相手に適切なコミュニケーションを取るため |
| **どう使われる** | 日常の連絡、面接調整、トラブル対応、関係構築 |

### この項目がないと何が起きるか
- 役務が誰に連絡すればいいか分からず、対応が遅れる
- 決裁者を把握できず、決定に時間がかかる
- 人柄を把握できず、コミュニケーションでミスマッチが起きる
- 「FSから聞いていない」とクレームになる

### 記載すべき内容

| 項目 | 説明 | 例 |
|------|------|-----|
| 担当者情報 | 日常の窓口となる人 | 氏名、役職、連絡先 |
| 担当者の人柄 | 性格、コミュニケーションスタイル | 明るい、せっかち、丁寧 |
| 連絡の好み | 連絡手段、時間帯 | 電話派、午後が繋がりやすい |
| 決裁者情報 | 最終決定する人 | 氏名、役職、関与度 |
| 現場責任者 | 面接官になる人 | 氏名、役職、重視点 |
| 連絡フロー | 誰にどう連絡するか | 基本は担当者、緊急時は決裁者 |

### 良い例
```
【採用担当者】
・氏名：山田花子
・役職：人事課主任
・連絡先：090-XXXX-XXXX（携帯）
・人柄：明るくテキパキ。結論から話すと好まれる。
・連絡：電話＞メール。午前中は会議が多く、13-17時が繋がりやすい。
・レスポンス：早い。当日中に返信がある。

【決裁者】
・氏名：鈴木太郎
・役職：施設長
・関与度：最終面接に同席、採用可否を決定
・特徴：じっくり考える。その場で即決しない。
・連絡：基本は山田さん経由。緊急時のみ直接連絡可。

【現場責任者】
・氏名：田中次郎
・役職：介護主任
・面接での役割：一次面接を担当
・重視点：人柄、チームワーク、体力
・特徴：現場を一番分かっている。候補者との相性を重視。

【連絡フロー】
日常連絡：山田さん → 重要事項は鈴木施設長にCC
面接調整：山田さん経由で田中主任と日程調整
緊急時：鈴木施設長に直接連絡可（山田さん不在時）
```

### NG例
```
担当：田中さん
いい人です
```
→ 情報として役に立たない

## 6. 担当者注意点

### 定義
**クライアント対応で注意すべきこと、避けるべきこと、配慮すべきことを記載する項目**

### 意図・目的

| 観点 | 説明 |
|------|------|
| **誰のため** | 役務 |
| **なぜ必要** | 役務がミスやトラブルを避け、良好な関係を維持するため |
| **どう使われる** | 日常のコミュニケーション、トラブル防止、関係構築 |

### この項目がないと何が起きるか
- 役務が地雷を踏んでクレームになる
- せっかく築いた関係が崩れる
- 「FSから聞いていない」と役務が困る
- 同じミスを繰り返す

### 記載すべき内容

| 項目 | 説明 | 例 |
|------|------|-----|
| 連絡タイミング | 避けるべき時間帯 | 月曜午前はNG、金曜は不在多い |
| 連絡手段 | 好まれる/避けるべき手段 | 電話派、メールは見落とし多い |
| 話し方 | 好まれるコミュニケーション | 結論から、丁寧に、データで |
| NGワード | 避けるべき話題・言葉 | 前の紹介会社の話、他社比較 |
| 価値観 | 先方が重視すること | 人柄重視、地域密着、長期就労 |
| 過去トラブル | 注意すべき過去の問題 | 以前の紹介会社でトラブルあり |
| 特別対応 | 必要な配慮 | 耳が遠い、忙しい、出張多い |

### 良い例
```
【連絡について】
・電話がベスト。メールは見落とし多い（2日返信なければ電話フォロー）
・月曜午前は会議。金曜は現場に出ていることが多い。
・13:00-17:00が最も繋がりやすい。

【コミュニケーション】
・結論から話す方が好まれる。前置きが長いとイライラされる。
・データや数字で説明すると納得しやすい。
・雑談は好まない（忙しいアピールあり）。用件のみ端的に。

【NGワード・避けるべき話題】
・前任の人材紹介会社の話（トラブルがあった様子で触れたくなさそう）
・他社との比較をあからさまにする（「A社より安い」など）
・「人手不足ですよね」という一般論（「うちは違う」と言われた）

【重視していること】
・候補者の人柄（スキルより人柄を優先）
・長く働いてくれるか（短期離職を非常に嫌う）
・地域への愛着がある人

【過去のトラブル】
・以前の紹介会社で、入社1週間で辞められたことがある
・そのため「すぐ辞めそうな人」に敏感。定着する理由を説明すること。

【特別対応】
・施設長は耳が遠い。対面・電話ではゆっくり大きな声で。
・面接は必ず施設見学とセット（候補者のミスマッチ防止のため）
・内定後は「改めてのお礼電話」を入れると喜ばれる。
```

### NG例
```
特になし
```
→ 注意点が全くないクライアントはいない

## 品質基準

### 最低限クリアすべき基準

| 項目 | 最低文字数 | 必須要素 |
|------|-----------|---------|
| 募集職種 | 100文字 | 職種、雇用形態、勤務地、給与 |
| 採用課題 | 80文字 | 背景、緊急度、決裁者 |
| 導入ポイント | 100文字 | 先方の言葉での理由 |
| 刺さったポイント | 80文字 | 具体的な提案と反応 |
| 先方関係者情報 | 150文字 | 担当者名、連絡先、人柄 |
| 担当者注意点 | 50文字 | 連絡の注意点 |

### 「良い引継ぎ」の判定基準

役務が読んで以下の質問に答えられるか：

- [ ] このクライアントは何を求めているか分かる
- [ ] なぜ当社と契約したか分かる
- [ ] 誰にどう連絡すればいいか分かる
- [ ] 避けるべきことが分かる
- [ ] すぐに顧客対応を始められる
