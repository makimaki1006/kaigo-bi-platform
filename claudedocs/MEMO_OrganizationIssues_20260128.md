# 組織課題メモ

**作成日**: 2026年1月28日

---

## 1. 組織構造の現状

```
【構造】
部長（1名）
    ↓
ユニット長（複数名・横並び）
    ↓
メンバー

【問題】
・構造がフラットすぎる
・ユニット長レイヤーが横に開いて平らに存在
・上司に対するレビュー体制がない
・各自が個別最適で動いてしまう
```

---

## 2. 方針・戦略の不在

| 現象 | 内容 |
|------|------|
| ユニット長の発言 | 「会社の方針がどこに向かっているかわからない」 |
| 主体性 | ない（受動的） |
| 結果 | 戦略的思考をせず、リソースが迷う |

---

## 3. フィードバック体制の欠如

**【横の連携】**
- ユニット長同士のフィードバック体制がない
- リーダー層同士の連携がない

**【縦の連携（ボトムアップ）】**
- 下からの報告・相談が少ない
- 報告する側の能力不足：
  - 「こういう風になります」という説明ができない
  - 資料をまとめる能力がない
  - 言語化・構造化ができない


---

## 4. レビューの機能不全

**【上層側の問題】**
- 現場の状況を把握していない
- 適切なレビューができない
- 「どうなってる？」という抽象的なコメントしかできない
- 判断基準が抽象度高いまま
- FBも抽象度が高く、「自分で考えろ」が多く、現場の「とはいえ時間の無い中で」とぶつかってワークしていない

**【報告側の問題】**
- 報告のレベルが低すぎて判断材料にならない

→ **お互いに原因がある**

---

## 5. 個別最適の弊害

```
連携なく各自がベストと思う方法で動く
        ↓
組織としての成果が出ない
        ↓
頑張ってるのに成果が出ない
        ↓
現場が疲弊する
```

---

## 6. 負のループ

```
相談しても意味ない
        ↓
やっても意味ないと感じる
        ↓
さらに疲弊
        ↓
相談しなくなる
        ↓
（ループ）
```

**【上からの反応】**
- 「自分で考えろ」という発言
- 抽象的なアドバイスばかり
- 具体的なアドバイスが少ない

---

## 7. 結論（根本原因）

```
両者（上層・現場）の言語化能力・構造化能力の不足
```

| 層 | 不足している能力 |
|----|-----------------|
| 上層 | 現場把握、具体的指示、判断基準の明確化 |
| 現場 | 報告能力、フォーマット化 など |
