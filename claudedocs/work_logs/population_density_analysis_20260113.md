## 4. 推奨フィルター設定

| 戦略 | 人口フィルタ | カバー密度帯 | 2025年受注率 |
|------|------------|-------------|-------------|
| **攻め** | 7万人以上 | 密度1万+/km²を100%カバー | **14.0%** |
| バランス | 5万人以上 | 密度8千+/km²を100%カバー | 11.2% |
| 緩め | 3万人以上 | 密度5千+/km²を100%カバー | 10.4% |

### 推奨

**人口7万人以上でフィルタ**
- 密度1万+/km²（超都市部）を100%カバー
- 2025年受注率: 14.0%（全体平均8.9%と比較して+5.1pt）
- 超都市部のみに絞ると受注率16.9%

## 6. 戦略的示唆

### やるべきこと

1. **アタックリストに人口フィルタを追加**
   - Salesforceリストビュー: `Population__c >= 70000`
   - 超都市部を優先ターゲット化

2. **人口1万人以下は除外検討**
   - 受注率に大きな差はないがエリア効率が悪い
   - 移動コスト・商談効率の観点から非効率

3. **営業トークの最適化**
   - 超都市部: 強気のトークが可能（受注率16.9%）
   - 地方: 引き続き期待値コントロール重視

### 注意点

- 人口フィルタは密度の「代替」であり完全一致ではない
- 人口7万人以上でも低密度エリアは含まれる（受注率が下がる要因）
- 密度セグメント単体の受注率と人口フィルタ適用後の受注率は異なる

## 9. 突合ロジック修正履歴

### 修正1: 都道府県の優先順位

**問題**: 住所に都道府県が含まれているのに、入力値の都道府県が優先されてしまう

```python
# 修正前（バグ）
# 住所: 兵庫県神戸市 + pref_input: 北海道 → 北海道神戸市（誤）

# 修正後
if detected_pref:  # 住所から検出された都道府県を優先
    final_pref = detected_pref
elif prefecture and not pd.isna(prefecture):
    final_pref = normalize_prefecture(prefecture)
```

### 修正2: 郡部の抽出

**問題**: e-Statデータは「岐南町」形式なのに「羽島郡岐南町」を抽出してしまう

```python
# 修正前（バグ）
# 住所: 羽島郡岐南町 → 羽島郡岐南町（マッチしない）

# 修正後
gun_pattern = re.compile(r'^.+?郡(.+?[町村])')
match = gun_pattern.match(address)
if match:
    return final_pref, match.group(1)  # 町村名のみ返す（岐南町）
```
