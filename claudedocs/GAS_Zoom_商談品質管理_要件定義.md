# GAS × Zoom × Gemini 商談品質管理システム 要件定義書

## 1. システム概要

### 1.1 目的
Medica事業部の営業メンバーの商談品質を自動的に分析・可視化し、継続的な品質改善を支援する。

### 1.2 システム構成図

```
┌─────────────────────────────────────────────────────────────────┐
│                        Google Apps Script                        │
│  ┌──────────────┐    ┌──────────────┐    ┌──────────────┐      │
│  │ 時間トリガー  │───▶│  メイン処理   │───▶│  出力処理    │      │
│  │  (1時間毎)   │    │              │    │              │      │
│  └──────────────┘    └──────┬───────┘    └──────┬───────┘      │
└──────────────────────────────┼──────────────────┼───────────────┘

           ┌───────────────────┼──────────────────┼───────────────┐
           ▼                   │                  ▼               ▼
```

## 2. 機能要件

### 2.1 Zoom API連携

#### 2.1.1 認証方式
- **Server-to-Server OAuth** を使用
- アクセストークンは1時間有効、GAS内でキャッシュ＆自動更新

#### 2.1.2 使用エンドポイント

| エンドポイント | 用途 | 頻度 |
|---------------|------|------|
| `POST /oauth/token` | アクセストークン取得 | トリガー毎 |
| `GET /users/{userId}/recordings` | ユーザーの録画一覧取得 | メンバー数×トリガー毎 |
| `GET /meetings/{meetingId}/recordings` | 録画詳細取得 | 新規録画数×トリガー毎 |
| `GET /recordings/{recordingId}/transcript` | 文字起こしVTT取得 | 新規録画数×トリガー毎 |

#### 2.1.3 取得データ
```javascript
// 録画一覧から取得する情報
{
  meeting_id: "ミーティングID",
  topic: "ミーティング名",
  start_time: "開始日時",
  duration: "所要時間（分）",
  host_email: "ホストメールアドレス",
  recording_files: [
    {
      id: "録画ファイルID",
      file_type: "TRANSCRIPT",  // 文字起こしファイル
      download_url: "ダウンロードURL"
    }
  ]
}
```

#### 2.1.4 API制限対応
- Zoom API Rate Limit: 30 requests/second
- GAS内でリクエスト間隔を調整（Utilities.sleep）
- 大量データ時はページネーション対応

### 2.2 スプレッドシート構成

#### 2.2.1 シート構成

| シート名 | 用途 |
|---------|------|
| `設定` | API認証情報、Slack Webhook URL等 |
| `対象メンバー` | 分析対象メンバーのZoomメール一覧 |
| `処理済み` | 分析済みミーティングID管理（重複防止） |
| `分析結果` | 全商談の分析結果蓄積 |
| `メンバー別サマリー` | メンバー別集計（自動計算） |
| `除外キーワード` | 除外する会議名キーワード |

#### 2.2.2 対象メンバーシート

| 列 | 内容 | 例 |

### 2.3 Gemini API連携

#### 2.3.1 モデル
- **gemini-2.0-flash** を使用
- 高速・低コストで大量処理に適合

#### 2.3.2 プロンプト構成

```javascript

[...省略（現行プロンプト全文）...]

【商談ログ】
```

#### 2.3.3 API呼び出し設定

| パラメータ | 値 |

## 3. 処理フロー

### 3.1 メイン処理フロー

```
1. トリガー起動（1時間毎）
   │
2. 設定シートから認証情報取得
   │
3. 対象メンバーシートからメンバー一覧取得
   │
4. 各メンバーについてループ
   │  │
   │  ├─ 4.1 Zoom API: 過去24時間の録画一覧取得
   │  │
   │  ├─ 4.2 処理済みシートと照合（重複スキップ）
   │  │
   │  ├─ 4.3 除外キーワードチェック（該当はスキップ）
   │  │
   │  ├─ 4.4 文字起こしVTT取得
   │  │
   │  ├─ 4.5 VTT→プレーンテキスト変換
   │  │
   │  ├─ 4.6 Gemini API: 商談分析
   │  │
   │  ├─ 4.7 レスポンスパース
   │  │
   │  ├─ 4.8 分析結果シートに書き込み
   │  │
   │  ├─ 4.9 処理済みシートにID追加
   │  │
   │  └─ 4.10 Slack通知送信
   │
5. 処理完了ログ出力
```

### 3.2 エラーハンドリング

| エラー種別 | 対処 |
|-----------|------|
| Zoom API認証エラー | トークン再取得してリトライ |
| Zoom API Rate Limit | 60秒待機してリトライ |
| 文字起こし未生成 | スキップ（次回トリガーで再試行） |
| Gemini APIエラー | 指数バックオフで3回リトライ |
| Gemini応答パース失敗 | エラーログ記録、Slack通知 |
| Slackエラー | ログ記録のみ（処理は継続） |

## 4. 環境設定

### 4.1 必要なAPI認証情報

| 項目 | 取得元 |
|-----|--------|
| ZOOM_ACCOUNT_ID | Zoom Marketplace > Server-to-Server OAuth App |
| ZOOM_CLIENT_ID | 同上 |
| ZOOM_CLIENT_SECRET | 同上 |
| GEMINI_API_KEY | Google AI Studio |
| SLACK_WEBHOOK_URL | Slack App > Incoming Webhooks |

### 4.2 Zoom App必要スコープ

```
recording:read:admin
user:read:admin
```

### 4.3 GAS実行時間制限対応

- GAS実行時間上限: 6分
- 対策:
  - メンバーを分割して複数トリガーで処理
  - または1トリガーで処理件数上限を設定（例: 10件/回）

---

## 5. 非機能要件

### 5.1 パフォーマンス
- 1商談あたり処理時間目安: 30秒〜60秒
- 1時間トリガーで最大10件程度を処理想定

### 5.2 コスト概算

| 項目 | 単価 | 月間想定 |
|-----|------|---------|
| Gemini API | 約$0.0001/1K tokens | 〜$5（5000件想定） |
| Zoom API | 無料（Pro以上プラン内） | $0 |
| GAS | 無料 | $0 |

### 5.3 セキュリティ
- API認証情報はGASスクリプトプロパティに格納
- 文字起こしデータはスプレッドシート外に保存しない
- Slack通知には顧客名フルを含めない（必要に応じて）
