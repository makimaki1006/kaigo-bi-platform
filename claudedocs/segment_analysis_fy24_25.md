## 1. エグゼクティブサマリー

### 最重要発見

1. **医療セクターは規模で切り分けるべき** — 「医療×医療法人」を一括で11.0%と見るのは誤り。小規模クリニック（30人以下）は**16.4%**、大病院（301人+）は**6.4%**。2.5倍の差。
2. **法人格が最強の判別因子** — 同じ施設形態でも法人格で3-18%の差が出る。特に介護セクターで顕著。
3. **介護×法人格不明（1,631件/3.3%）が全体の足を引っ張る最大要因** — Account.Nameは施設名のみで法人格推論不可。法人番号（72%に存在）からの外部照合で解決可能。

## 7. 介護×法人格不明（1,631件）問題

### 問題の規模

- 初回商談10,111件のうち**16.1%** を占める
- 受注率3.3%（全体平均9.0%の1/3以下）
- **このセグメントを回避するだけで全体受注率が+2.0pt改善**

### なぜ法人格が不明か

| 項目 | 結果 |
|------|------|
| Account.Name | 施設名のみ（「○○デイサービス」等）。法人格を含まない → **推論不可** |
| 親Account | 462件に親あり。うち法人格が分かるのは**64件のみ（3.8%）** |
| 法人番号 | **1,218件（72%）に法人番号あり** → 外部照合で解決可能 |
| CustomerSegment | 77%が「02-D 過去失注先」 |

### 推論方法の比較

| 方法 | カバー率 | 実現性 | 精度 |
|------|---------|--------|------|
| Account.Nameから抽出 | 0.1% | 不可 | - |
| 親Accountの法人格 | 3.8% | 即時 | 高 |
| **法人番号→国税庁API** | **72%** | **要API登録** | **高** |
| 手動入力 | 100% | 非現実的 | 高 |

### 法人番号→国税庁API照合の提案

**国税庁法人番号公表サイトAPI** を使用して法人番号から法人名を取得し、法人名から法人格を抽出する。

- 対象: 1,218件（法人番号あり）
- 期待される効果: 法人格不明1,631件のうち約72%の法人格を特定
- 特定後: 法人格ごとに受注率でリスト優先度を再計算可能

### 法人格判明時の推定効果

親Accountから法人格が判明した64件では、法人格ありの介護セクターと同じ傾向が確認された：

| 親の法人格 | 件数 | 受注率 | 法人格判明介護の受注率 |
|-----------|------|--------|-------------------|
| 株式会社 | 20件 | **40.0%** | 15.4% |
| 医療法人 | 9件 | **22.2%** | 10.2% |
| 社会福祉法人 | 28件 | **10.7%** | 17.9% |

→ **法人格が分かれば、高受注率（株式会社）と低受注率を選別できる。**

## 10. 法人番号→法人格の一括推論（実施済み）

### 背景

介護×法人格不明（1,631件/3.3%）が全体の足を引っ張る最大要因だが、Account.Nameは施設名のみで法人格を含まない（推論成功率0.1%）。一方、法人番号（CorporateNumber__c）は72%のAccountに入力されている。

### 手法

1. Salesforceから `LegalPersonality__c` が空 かつ `CorporateNumber__c` が入っているAccountを抽出
2. 国税庁法人番号公表サイトの全件CSV（約500万法人）と法人番号で突合
3. 国税庁の法人名（「株式会社○○」「社会福祉法人○○」等）から法人格を正規表現で抽出
4. Salesforce更新用CSVを生成

### データソース

| 項目 | 詳細 |
|------|------|
| ファイル | `00_zenkoku_all_20260227.csv`（Unicode版） |
| ダウンロード元 | https://www.houjin-bangou.nta.go.jp/download/zenken/ |
| ファイルサイズ | 1.2GB（約575万行） |
| ダウンロード日 | 2026-02-27 |
| 登録 | 不要（無料・即時） |

### 実行結果

| 項目 | 件数 |
|------|------|
| 法人格不明Account（法人番号あり） | 50,830件 |
| 国税庁CSVとマッチ | 41,760件（98.9%） |
| **法人格の推論成功** | **47,411件（93.3%）** |
| 推論不可 | 2,783件 |
| 国税庁にマッチなし | 636件 |

※ 47,411件 > 41,760件 なのは、一部Accountに複数の法人番号→法人名マッチがあるため。重複除去後の更新対象Account数が47,411件。

### 推論不可のパターン（2,783件）

法人名は取得できたが法人格パターンにマッチしなかったもの。例：
- `社会保険労務士法人○○` → 「社会福祉法人」ではなく「士業法人」
- `国立大学法人○○` → 独立行政法人系
- `更生保護法人○○` → 特殊法人
- `独立行政法人○○` → 国系法人
- 地方自治体（`穴水町` 等）

### 更新手順（未実施・要確認）

**LegalPersonality__cへの直接書き込みはリスクがあるため、以下の確認を推奨：**

1. **サンプル検証**: 推論結果の正確性をランダム20-50件で目視確認
2. **既存値との矛盾チェック**: LegalPersonality_CompanyName__c や LegalPersonality_AccountName__c に値がある場合、推論値と一致するか確認
3. **バックアップ**: 更新前に現在のLegalPersonality__c値をCSVエクスポート
4. **段階的更新**: まず100件で試行→問題なければ全件実行
5. **更新後の検証**: 商談分析を再実行し、法人格不明件数の減少と受注率変化を確認
