## 4. 実装ロードマップ

### Phase 1: 詳細HTMLパーサー開発（最優先）
- 36種別の詳細HTMLから構造化データを抽出するPythonスクリプト
- tableGroup-4（従業者）とtableGroup-5（サービス内容・加算）を優先パース
- パース結果をCSVに出力 → Tursoにアップロード

### Phase 2: バックエンド拡張
- 新テーブル（詳細データ）のAPI追加
- サービス種別パラメータに応じた動的レスポンス（利用可能KPIリスト返却）
- 未活用外部統計テーブル（8テーブル）のAPIエンドポイント追加

### Phase 3: フロントエンド サービス種別コントロール
- `useServiceTypeConfig` フック: 選択種別に応じて可視化項目を動的制御
- 各ページで `config.isAvailable('kpi_name')` による条件付きレンダリング
- データなし項目は「このサービス種別では非該当」と明示

### Phase 4: 加算・減算分析ページ（新規）
- サービス種別ごとの加算取得率一覧
- 都道府県別加算取得率比較
- 加算取得パターン分析（クラスタリング）

### Phase 5: 外部データ統合強化
- 未活用8テーブルのAPI+フロントエンド実装
- 人口×施設密度、介護需要×供給ギャップ等のクロス分析
- 求人給与統計との比較分析

```typescript
  fillRate?: number; // データ充填率（%）
  note?: string;     // 非該当理由等
}

interface ServiceTypeConfig {
  code: string;
  name: string;
  category: 'visit' | 'day_care' | 'residential' | 'community' | 'care_plan' | 'other';
  kpis: Record<string, KpiAvailability>;
  kasanList: string[]; // この種別で適用される加算名リスト
}
```

### 5.2 ページ側の制御例

```tsx
// 各ページで
const config = useServiceTypeConfig(selectedServiceCode);

return (
  <>
    {/* 全種別共通: 施設数 */}
    <FacilityCountCard data={data} />

    {/* 通所・入所系のみ: 定員充足率 */}
    {config.isAvailable('capacity_utilization') && (
      <CapacityUtilizationChart data={data} />
    )}

    {/* 非該当項目は理由を表示 */}
    {!config.isAvailable('capacity_utilization') && (
      <NotApplicableMessage
        kpi="定員充足率"
        reason={config.getReason('capacity_utilization')}
      />
    )}
  </>
);
```
