# リード施策ROI分析レポート -- Lead起源の真の貢献率と受注効率の解明

**分析日**: 2026年3月10日
**データ期間**: Opportunity CloseDate 2025年10月 - 2026年3月 / Lead 142,693件（2026-03-05時点）
**分析対象**: Lead 142,693件 / Opportunity 3,289件 / Account 253,859件

---

## 1. エグゼクティブサマリー

**結論: リード施策の真の貢献率は38.2%であり、従来計測の0.34%は構造的な計測不備による大幅な過小評価であった。**

主要な発見は以下の3点である。

- **リード施策（ハローワーク + 有料媒体）は受注の38.2%に関与しており、従来計測（0.34%）の112倍の貢献度がある。** Salesforce標準のConvertedOpportunityIdが全件空であるという構造的問題が原因で、リード施策の効果が正しく計測されていなかった。
- **ハローワーク(HW)と有料媒体の情報が両方揃ったリードは、HWのみと比較して成約率が1.6倍（4.9%→7.9%）、有料媒体のみと比較して3.0倍（2.6%→7.9%）。** 「誰に何を言えばいいかわかる」+「お金を出す気がある」の2条件が揃うことで、受注効率が向上する。ただし(4)両方カテゴリのサンプルサイズ（受注15件）は小さく、統計的に頑健な結論を導くには追加データが必要。
- **HWデータ更新の遅延は受注効率の低下と相関がある。** 2026年1月にHW更新が滞った期間、HW関与率は25.9%まで低下し受注27件と4ヶ月間で最低となった。2月に更新を再開したことで40.0%に回復、200%達成の土台となった。ただし、1月の低調は営業日数（19日間）や年始の商習慣など複数の要因が重なっている可能性があり、HW更新遅延のみに帰因すべきではない。

## 2. 分析の背景と目的

### 前回分析の問題提起

2月の200%達成分析において、Salesforce標準のLeadコンバージョン追跡（ConvertedOpportunityId）を用いたリード貢献率は**0.34%**と算出された。しかし、現場の営業活動において「リスト施策がこれほど貢献していないはずがない」という感覚的な乖離があった。

### 構造的問題の発見

調査の結果、**ConvertedOpportunityIdが全件空**であることが判明した。これはLeadのコンバージョン時にOpportunityへの紐付けが行われていないことを意味し、Salesforce標準の追跡機能が機能していなかった。

### 本分析の目的

ConvertedAccountId、電話番号、会社名を用いた多段階マッチングにより、Lead起源のOpportunityを正しく特定し、リード施策の真の貢献率と各施策カテゴリ別の受注効率を定量的に明らかにすることを目的とする。

## 3. 分析手法

### 4段階マッチング

Lead→Opportunityの直接追跡ができないため、Lead→Account→Opportunityの間接経路を中心とした4段階マッチングを実施した。

| 段階 | マッチング手法 | マッチ件数 | 信頼度 |
|------|-------------|-----------|--------|
| 第1段階 | ConvertedAccountId（SF標準リンク） | 69件 | 最高 |
| 第2段階 | 電話番号マッチ（Account全15列 → Lead4列） | 5件 | 高 |
| 第3段階 | 会社名完全一致（法人格除去後） | 5件 | 高 |
| 第4段階 | 会社名部分一致（5文字以上制限） | 31件 | 中 |
| -- | マッチなし | 115件 | -- |
| **合計** | -- | **225件（受注Opportunity）** | -- |

マッチなしの115件（51.1%）は、紹介や直接営業などリスト施策を経由せずに受注した案件と考えられる。

## 6. なぜ(4)「両方」が圧倒的に効率が良いのか

(4)「HW+有料媒体 両方」のリードが他カテゴリを大きく引き離す理由は、2つの情報源がそれぞれ異なる意思決定材料を提供するためである。

### ハローワークデータが提供する情報

| 情報 | 営業活動への効果 |
|------|----------------|
| 代表者名・担当者名・役職 | 決裁者に直接アプローチ可能 |
| 募集理由（欠員/増員/新規事業） | 採用の緊急度がわかる |
| 募集職種 | 弊社サービスとのフィット判定 |
| 従業員数・資本金 | ターゲットセグメント選別（30～150名の株式会社） |
| 法人番号 | 名寄せ・重複排除の精度 |

### 有料媒体データが提供する情報

| 情報 | 営業活動への効果 |
|------|----------------|
| 有料媒体を使っている事実 | 採用にお金をかけている = 支払い能力・意欲の証拠 |

### 両方揃った場合の相乗効果

> **「誰に何を言えばいいかわかる」（HW）+「お金を出す気がある」（有料媒体）**

この2条件が揃うことで、架電率91.7%、商談化率83%、成約率7.9%という全指標でトップの数値が実現される。営業担当者は「この会社は介護職の欠員補充で困っていて、すでにタウンワークに出稿している。代表の田中様に直接お電話すれば話が早い」といった精度の高いアプローチが可能になる。

## 7. (1)「マスターデータ残骸」のアポ率が高いのに成約率が低い理由

(1)マスターデータ残骸は、一見するとアポ率6.49%（全カテゴリ最高）で好成績に見える。しかし成約率は1.1%（全カテゴリ最低）であり、ファネル後半で急激に効率が落ちる。

### 数値で見るファネル崩壊

| ファネル段階 | 件数 | 次段階への通過率 |
|------------|------|----------------|
| CV（コンバージョン） | 974件 | -- |
| 商談化 | 180件 | **18%** |
| 成約 | 11件 | 1.1% |

CV974件のうち商談化はわずか180件（18%）であり、(2)HWの65%、(4)両方の83%と比較して著しく低い。

### 根本原因: 情報不足による提案精度の低下

- 担当者名・役職が不明 → 「ご担当者様」で架電するため接触効率が低い
- 募集理由・職種が不明 → 提案内容が的確にならない
- 従業員数が不明 → セグメント判定ができず、ターゲット外企業にも時間を消費

### 補足: 「その他」に隠れたHW起源リード

(1)に分類された約431件は、実際にはHW起源であるがHellowork_*フィールドが未入力のリードである可能性がある。HWデータ付与のカバレッジを上げることで、(1)から(2)へ再分類され、実質的な受注効率の改善が見込める。

---

## 8. (3)有料媒体のみの成約率が(2)HWより低い理由

(3)有料媒体のみのリードは、成約率2.6%であり、(2)HWの4.9%を下回る。有料媒体に出稿している企業は支払い能力があるはずだが、なぜ成約率が低いのか。

### 原因1: 従業員数が無差別

有料媒体のスクレイピングデータには従業員数が含まれないケースが多い。その結果、従業員数500名以上の大企業も含まれてしまい、代表者に辿り着けない商談が増える。弊社の強みが発揮される「株式会社 x 30～150名」のセグメントに絞れていない。

### 原因2: 担当者名が取れない

有料媒体の求人広告には担当者の個人名が記載されないことが多い。「ご担当者様」での架電は、HWデータにある「選考担当者: 田中太郎（人事部長）」での架電と比較して、接触率が大きく低下する。

### 解決策

| 対策 | 期待効果 |
|------|---------|
| バフェットコード社のデータで従業員数を補完 | セグメント2軸（株式会社 x 30～150名）でのフィルタリングが可能になる |
| HWデータとの突合を徹底 | (3)から(4)へ移行するリードが増え、成約率が2.6%→7.9%に向上 |

従業員数30～150名の株式会社であれば、代表者に直接繋がりやすいため、担当者名がなくても代表者商談の確率が高まり、成約率の低下を相殺できる。

## 9. 月別トレンドとHW更新の影響

| 月 | 受注件数 | HW関与率 | 「両方」件数 | 備考 |
|----|---------|---------|------------|------|
| 11月 | 45件 | 33.3% | 5件 | -- |
| 12月 | 52件 | 28.8% | 2件 | -- |
| 1月 | 27件 | 25.9% | 1件 | HWデータ更新滞り → HW関与率が最低 |
| 2月 | 45件 | 40.0% | 6件 | HW更新再開 → 200%達成 |

1月にHWデータ更新が滞った期間、HW関与率は25.9%まで低下し、受注件数も27件と4ヶ月間で最低となった。2月にHW更新を再開したことでHW関与率は40.0%に回復し、「両方」リードも6件に増加、200%達成の基盤となった。

**留意点**: 1月の受注件数低下はHW更新遅延だけでなく、営業日数の少なさ（19日間、年始休暇含む）や年始の商習慣の影響も受けている。ただし、営業日数の減少率（約10%減）に対して受注件数の減少率（約40～47%減）は大きく、営業日だけでは説明がつかない構造的な要因がある可能性が高い。

## 10. 2月200%達成の因果メカニズム（前回分析との統合）

前回の2月200%達成分析では、「再商談 x 代表者戦術による+11件」という**戦術層**の効果が主な発見であった。しかし今回の分析により、その戦術が成功した土台には**データ基盤 → セグメント → 優先度付け**という3層の仕組みがあることが判明した。

### 成約率の積み上げ構造（仮説モデル）

以下は各施策の効果を段階的に示した**概念モデル**であり、各層の増分は個別に検証されたものではない。実際のデータは(1)～(4)のカテゴリ別成約率として計測されており、層間の分離は推論に基づく。

```
情報なし（(1)残骸）:               1.1%  ← 実測値
  + セグメント2軸:                → 4～5%  ← 推定
  + 有料媒体フラグ:                → 6～7%  ← 推定
  + HW突合（募集理由・担当者名）:    → 8～9%  ← (4)両方の実測値7.9%と整合
  + 再商談 x 代表者戦術:           → 約10%  ← 2月実績からの推定
```

前回分析で可視化できていたのは最上層の「再商談 x 代表者戦術」のみであった。今回の分析で、その下にデータ基盤の層が存在し、カテゴリ間の成約率格差（1.1%→4.9%→7.9%）として観測されることが判明した。

**2月200%達成は、戦術の巧みさだけでなく、HWデータ更新再開によるデータ基盤の回復が不可欠な前提条件であった。**

## 11. HWデータ更新を止めるとどうなるか

HWデータの定期更新を停止した場合、(2)HWカテゴリのリードが徐々に(1)残骸相当に劣化する。その影響を定量的に示す。

| 指標 | 更新あり（(2)HW） | 更新なし（(1)残骸相当） | 劣化倍率 |
|------|-------------------|----------------------|---------|
| 商談化率 | 65% | 18% | **3.6分の1** |
| 成約率 | 4.9% | 1.1% | **4.5分の1** |
| Lead1件あたり売上 | 430円 | 115円 | **3.7分の1** |

HW更新を停止すると、64,830件のHWリードが情報劣化し、成約率は4.9%から1.1%へ低下する。受注金額に換算すると、HW起源の2,786万円のうち約2,160万円が失われるリスクがある。

## 12. 今後の戦略

### 戦略の3本柱

**主軸: ハローワーク定期更新の継続**

担当者名・役職・募集理由・職種・従業員数を継続的に付与する。これだけで成約率は1%から5%へ（5倍）向上する。最もコストパフォーマンスが高い施策であり、停止は許容されない。

**攻め: 有料媒体の積極開拓**

有料媒体への出稿実績は「資金力の証拠」である。HWデータと突合すれば成約率は8～10%に達し、情報なしリードの26倍の効率となる。

**補完: バフェットコード社データによる従業員数付与**

有料媒体のみのリードにも従業員数を補完することで、セグメント2軸（株式会社 x 30～150名）でのフィルタリングが可能になる。HW突合なしでも成約率4～5%まで引き上げが見込める。

### 打ち手の優先順位

| 優先度 | アクション | 期待効果 |
|--------|----------|---------|
| 最優先 | HWデータ定期更新を継続 | 成約率4.9%の維持（止めると1.1%に低下） |
| 高 | 有料媒体 x HW突合を毎回実施 | 「両方」リード増加 → 26倍効率の母数拡大 |
| 高 | バフェットコードで従業員数補完 | 有料媒体のみリードのセグメント精度向上 |
| 中 | 有料媒体からの積極的リード化 | (4)の母数拡大 |
| 監視 | 担当者名なしリードの接触率追跡 | 悪化傾向が出れば対策を検討 |

### 懸念事項

- 有料媒体のみのリードは担当者名が取れない可能性が高い
- ただし従業員数30～150名の株式会社であれば代表者に直接繋がりやすいため、担当者名不在の影響は相殺可能

## 13. 分析の限界と留意事項

本分析は以下の制約のもとで実施されており、結果の解釈にあたって留意が必要である。

### データ・手法上の制約

| 制約事項 | 影響 |
|---------|------|
| ConvertedOpportunityIdが全件空 | Lead→Opportunityの直接追跡ができず、間接マッチングに依存 |
| AccountId経由の間接追跡 | Lead→Account→Opportunityの経路であるため、Account統合等による誤マッチの可能性 |
| マッチなし115件（51.1%） | 紹介・直接営業等でリスト施策を経由していない受注の可能性 |
| 部分一致マッチ31件 | 施設名の類似による誤マッチを含む可能性あり（5文字以上制限で緩和） |
| データ期間の限定 | Opportunity CloseDate 2025年10月～2026年3月の6ヶ月間に限定 |
| Lead件数の時点 | 2026年3月5日時点の142,693件であり、過去に削除されたLeadは含まれない |

### 統計的限界

| 制約事項 | 影響 |
|---------|------|
| (4)両方カテゴリの受注サンプルn=15 | 1件の増減で成約率が±0.5ポイント変動。カイ二乗検定等の有意差検定は未実施 |
| カテゴリ間の交絡因子 | 架電率の差（26.6%～91.7%）は営業優先度を反映。データ品質の効果と営業リソース配分の効果を分離できていない |
| リード鮮度の効果 | HWリードは全て2025年11月以降（最大5ヶ月）、残骸は2025年7月以前。成約率の差にはデータ種別だけでなく情報の新鮮さも寄与している可能性 |

### 因果関係と相関の区別

本レポートで示されたカテゴリ間の成約率格差は**相関関係**であり、厳密な因果関係を主張するものではない。例えば、(4)両方カテゴリの成約率7.9%がデータ品質のみに起因するとは限らず、以下の代替仮説も排除できない。

- 営業担当者が情報の多いリードを優先的に架電している（選択バイアス）
- HWリードは情報が新しく、連絡先が有効な確率が高い（鮮度効果）
- 有料媒体に出稿する企業は業界全体で成約しやすい特性がある（共通原因）

ただし、現場の営業活動において「担当者名があるから話が通じやすい」「募集理由がわかるから提案が刺さる」という質的な裏付けがあり、データ品質→提案精度→成約率の因果経路は蓋然性が高いと考えられる。

## 14. 補足: Lead不在案件（115件）の特性

マッチなし115件（Lead不在）は、リスト施策を経由せずに受注した案件と推定される。注目すべき点として、Lead不在案件の平均受注単価は約100万円であり、リスト経由案件（65～93万円）より高い傾向がある。

これは、紹介・直接問い合わせ・既存顧客からの追加受注などの非リスト経路が、高額案件を生んでいる可能性を示唆する。リスト施策のROIを評価する際、リスト施策が低単価案件を量産しているのか、あるいは他チャネルの高単価案件に対して数量で補っているのか、という構造を意識する必要がある。

**本レポートの戦略提言（リスト施策強化）は、Lead不在チャネルとの相対評価ではなく、リスト施策内でのカテゴリ間効率差に基づいている。**
