# 失注後フォローアップ電話 チェックリスト

**作成日**: 2026-02-13
**目的**: 失注先に「上司」として電話し、商談の振り返りと改善ポイントを収集する
**連動**: 失注分析Gemプロンプト（Layer 1/2/3判定の検証 + プロダクト/サービス軸のリアルな顧客声の収集）

---

## 1. 電話の位置づけ

### なぜやるか

- 文字起こし分析だけでは**顧客視点**が欠ける
- Gemプロンプトで判定したLayer分類が正しいか**検証**できる
- 特にプロダクト/サービス軸の問題は**顧客に聞かないとわからない**
- 「防げた失注」なのか「最初から無理だった」のかを**顧客の口から確認**する

### 誰が電話するか

- **商談担当者の上司**（またはそのふりをする人）
- 「○○の上司の△△と申します。先日はお時間いただきありがとうございました」

### タイミング

- 失注確定後**1〜3営業日以内**（記憶が新鮮なうちに）
- 遅くとも1週間以内

### 所要時間

- **5〜10分**を目安。長引かせない

---

## 2. 電話の導入スクリプト

```
「お世話になっております。メディカの△△と申します。
 先日○○（担当者名）が商談させていただいた件で、
 上司としてお礼のご連絡をさせていただきました。

 お忙しいところ恐れ入りますが、2〜3分ほどお時間よろしいでしょうか。

 今回はご縁がなかったのですが、○○のためにも今後の改善に活かしたく、
 率直なご意見をいただけると大変ありがたいのですが…」
```

**ポイント**:
- 売り込みではないことを明確にする
- 「部下のために」という姿勢を見せる
- 「率直に」を強調する（本音を引き出す）

---

## 3. チェックリスト（質問項目）

### Phase 1: Layer判定の検証（最初に聞く）

**目的**: Gemプロンプトで判定したLayer分類が正しいか確認

| # | 質問 | 検証対象 | 回答から読み取ること |
|---|------|---------|-------------------|
| 1-1 | 「今回お見送りになった一番の理由は何でしたか？」 | 全Layer | オープンクエスチョン。顧客が最初に言うことが本音に最も近い |
| 1-2 | 「採用のご状況は現在も変わらずですか？」 | Layer 1検証 | ニーズの有無を確認。「もう充足した」→L1、「まだ困っている」→L2の可能性 |
| 1-3 | 「他社さんでお決めになられましたか？」 | Layer 3検証 | 競合負けかどうか。「はい」→競合分析へ、「いいえ、今は見送り」→L1 or L2 |

### Phase 2: 営業行動の振り返り（Layer 2が疑われる場合）

**目的**: 因果的CPの不足が顧客にも感じられていたか確認

| # | 質問 | 対応するCP | 回答から読み取ること |
|---|------|-----------|-------------------|
| 2-1 | 「○○（担当者）の説明でわかりにくかった点はありましたか？」 | 全般 | 商談の質への率直な評価 |
| 2-2 | 「御社の採用課題について、もう少し詳しくお聞きすべきだった点はありますか？」 | CP05(課題ヒアリング深さ) | 掘り下げ不足を顧客が感じていたか |
| 2-3 | 「同業他社さんの事例などは参考になりましたか？あるいは不足していましたか？」 | CP23(他社事例) | 事例の説得力、不足感 |
| 2-4 | 「当社のサービスが御社のお役に立てるイメージは持てましたか？」 | CP20(リフレーム) | 価値提案が伝わったか |
| 2-5 | 「ご検討いただくうえで、もっと欲しかった情報やデータはありましたか？」 | CP12(求人データ) | データ不足の自覚 |

### Phase 3: プロダクト/サービス軸の検証（最重要）

**目的**: 営業スキルではなく、サービス自体の問題を検出する

| # | 質問 | 検出対象 | PM/商品企画へのフィードバック |
|---|------|---------|--------------------------|
| 3-1 | 「料金について率直にどう感じられましたか？」 | 料金体系 | 高い/安いだけでなく、構造（固定/成果報酬、初期費用等）の問題を探る |
| 3-2 | 「当社のサービス内容で、もう少しこういう部分があればと思った点は？」 | サービス範囲 | 顧客が期待していたのに提供できなかったもの |
| 3-3 | 「他社さんと比べて、当社の強み・弱みはどう映りましたか？」 | 競合差別化 | **競合に負けた場合は必ず聞く。** 何で負けたかの具体情報 |
| 3-4 | 「契約の条件面（期間、解約条件等）で気になった点はありましたか？」 | 契約条件 | 契約のハードルが高すぎないか |
| 3-5 | 「人材紹介サービス全般に対するイメージを教えていただけますか？」 | 品質懸念/市場 | 業界全体への不信感 or 過去のネガティブ経験の有無 |

### Phase 4: 競合情報（他社に決めた場合のみ）

**目的**: 具体的な競合情報を収集する

| # | 質問 | 収集情報 |
|---|------|---------|
| 4-1 | 「差し支えなければ、どちらの会社さんにされたか教えていただけますか？」 | 競合社名 |
| 4-2 | 「そちらに決められた一番のポイントは何でしたか？」 | 決め手（価格/品質/関係性/実績 等） |
| 4-3 | 「当社にそのポイントがあれば、ご検討いただけた可能性はありましたか？」 | 逆転可能性の検証 |

### Phase 5: クロージング

```
「大変参考になるお話をいただきありがとうございます。
 ○○（担当者）にもしっかり伝えて、今後のサービス改善に活かしてまいります。

 もし今後、採用のご状況が変わられた際には
 ぜひまたお声がけいただければと思います。
 本日は貴重なお時間をいただきありがとうございました。」
```

## 5. 集計・活用方法

### 月次レビューでの活用

| 集計項目 | 計算方法 | 活用先 |
|---------|---------|-------|
| Layer判定の修正率 | layer_verifiedが"changed"の割合 | Gemプロンプトの精度改善 |
| プロダクト起因率 | product_issue_verifiedが"none"以外の割合 | PM/商品企画への改善要望 |
| 競合負け先TOP3 | competitor_nameの頻度順 | 競合分析・差別化戦略 |
| Winback可能性 | winback_potentialが"high/medium"の件数 | 再アプローチリスト |
| 最頻失注理由 | customer_reasonの分布 | 営業/プロダクト双方の改善優先度 |

### Gemプロンプトへの還元

フォローアップ電話で得た情報をGemプロンプトの分析結果と突合し:
- **Layer判定が頻繁に修正される場合**: Gemプロンプトの判定基準を見直す
- **プロダクト課題が多い場合**: Gemプロンプトのプロダクト軸の検出精度を上げる
- **顧客の言葉と分析の乖離が大きい場合**: 因果的CPの定義を再検討する

## 6. 注意事項

1. **売り込みにならないこと**: あくまで「振り返り」「改善のため」。少しでも売り込み感が出ると本音が聞けなくなる
2. **担当者を批判しないこと**: 「○○が至らなかったかもしれませんが」程度は良いが、担当者を悪く言わない
3. **全部聞かなくていい**: 相手の状況に応じて質問を取捨選択する。5分で終わっていい
4. **記録は即座に**: 電話直後に記録する。記憶が薄れると顧客の言葉を正確に再現できない
5. **Layer 1（ニーズなし）の場合は短くていい**: Phase 1で「そもそもニーズがない」と判明したら、Phase 2-3は省略可能
6. **競合に決めた場合は Phase 4 を必ず実施**: 競合情報は非常に価値が高い
