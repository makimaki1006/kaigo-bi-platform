# 因果関係 vs 相関関係: チェックポイントの再分類

**作成日**: 2026-02-12
**前提**: 34チェックポイントの統合分析結果（198件上位 vs 225件下位）を再解釈する

---

## 0. 問題提起

先行するチェックポイント分析は「出現率の差分」で★必須/◎重要/●非差別化を判定した。
しかし、これは**相関**であって**因果**ではない。

```
× 「承知フレーズを増やせば受注率が上がる」
○ 「良い商談では自然と承知フレーズが出る」（=結果の指標）

× 「顧客料金質問が多いと負ける」
○ 「価値提案が弱いと顧客は料金しか聞くことがない」（=上流の問題の症状）
```

**分けるべき2つ**:

| 分類 | 意味 | 施策の方向性 |
|------|------|-------------|
| **因果的チェックポイント** | 意図的に実行すべき行動。やるかやらないかは営業の選択 | トレーニング対象。「これをやれ」 |
| **相関的チェックポイント** | 良い/悪い商談の結果として自然に現れる現象 | モニタリング指標。「これが出ていれば良い兆候/悪い兆候」 |

## 1. 因果的チェックポイント（意図的に通るべきゲート）

**判定基準**: 営業が「やる/やらない」を選択できる。事前準備や意識的なスキルで制御可能。

### ★ 因果的 × 差分大（最優先で訓練すべき）

| CP | 統合差分 | なぜ因果と言えるか | 施策 |
|----|---------|-------------------|------|
| **CP20_人事部リフレーム** | +20.9% | 「人事部のように」というフレーズは意図的にしか出ない。偶然言う人はいない。事前に準備しているかどうかの問題 | スクリプト化 + ロープレ |
| **CP23_他社事例** | +17.2% | 事例を知っていて意図的に提示するアクション。準備していないと出せない | 事例カード整備 + 商談前準備 |
| **CP12_求人データ** | +15.3% | 求人倍率等のデータを事前に調べて提示する行動。準備が前提 | データシート整備 + 準備シート |
| **CP07_決裁確認** | +14.0% | 「決裁はどなたですか？」は営業が意識的に聞く質問。聞かない人は聞かない | ヒアリング項目に追加 |
| **CP04_紹介経験確認** | +10.9% | 「人材紹介から採用されたご経験は？」は北尾のキラー質問。意識して使う技法 | スクリプト化 |
| **CP05_課題ヒアリング（深さ）** | 有無は+1.9%だが**深さが因果** | 全員がやっている（上位82.3%/下位80.4%）。しかし**深さ**は営業の意図的な選択。表面的な「困ってますか？」→「はい」で終わるか、3段階掘り下げるかは営業の技量と意図の問題 | 3段階掘り下げフレームワーク |

**課題ヒアリング「深さ」の定義:**

```
レベル1（表面的）: 「何かお困りのことはありますか？」→「はい、人が足りないです」→ 次の話題へ
  → これは全員やっている。差がつかない。

レベル2（構造的）: 「人が足りない」→「具体的にどの職種が？」→「何名くらい不足？」→「いつから募集されてます？」
  → 課題の構造（何が・どれだけ・いつから）を把握する。上位はここまでやる。

レベル3（因果的）: 構造把握の上で→「なぜ採れないとお考えですか？」→「今の手法で何が課題ですか？」→「それが続くとどうなりますか？」
  → 課題の原因と影響まで掘り下げる。SPIN selling の Problem → Implication に相当。
  → ここまで掘り下げると、顧客自身がニーズを言語化し、Q2のフレーミングが刺さりやすくなる。
```

**なぜ「深さ」は因果と言えるか:**
- 表面で終わるか、3段階掘り下げるかは**営業の意図的な選択**
- 準備（どんな質問を用意するか）と技量（顧客の回答を受けて深掘りできるか）で制御可能
- 「全員やっている」のは事実だが、**深さの違い**が後のQ2フレーミングの効果に直結する
- 深いヒアリング → 顧客の課題言語化(CP60)が増える → フレーミングが的を射る → 受注率上昇

**→ この6つは「やるかやらないか」「どこまでやるか」の問題。トレーニングで直接改善可能。**

### ◎ 因果的 × 差分中（訓練対象だが優先度は2番手）

| CP | 統合差分 | なぜ因果か | 備考 |
|----|---------|-----------|------|
| **CP13_調査提示** | +10.0% | 「調査によると…」は意図的なデータ引用行動 | データ根拠/分と重複する面あり |
| **CP41_キャンペーン** | +8.2% | キャンペーンの提示は営業の判断で行う | ただし使える場面が限定的 |
| **CP30_機会損失** | +6.1% | 「このまま放っておくと…」は意図的な危機感醸成 | 示唆質問(CP31)と類似 |
| **CP44_プラン提示** | +6.0% | 具体プランの提示は営業の判断 | ほぼ全員やっている(上位94%/下位88%) |

### 因果的だが非差別化（全員やっている基本動作）

| CP | 上位 | 下位 | 差分 | 意味 |
|----|------|------|------|------|
| CP03_現状手法確認 | 98.0% | 98.2% | -0.2% | 全員がやる基本ヒアリング。やらないと始まらない |
| CP24_サービス説明 | 96.5% | 97.8% | -1.3% | 全員がやる。差がつくのは説明の「質」 |
| CP01_職種確認 | 86.9% | 80.9% | +6.0% | 基本ヒアリング |

※ CP05_課題ヒアリングは「有無」では非差別化だが、「深さ」が因果的であるため上位セクションに移動。

**→ これらは「やるかどうか」では差がつかない。「どのタイミングで、どう組み立てるか」が真の差別化。**

## 2. 相関的チェックポイント（商談の健全性を示す温度計）

**判定基準**: 営業が直接コントロールしにくい。顧客の反応や商談の流れの結果として現れる。

### 顧客側の反応（= 営業がコントロールできない結果指標）

| CP | 統合差分 | なぜ相関か | 正しい読み方 |
|----|---------|-----------|-------------|
| **CP60_顧客課題言語化** | **-6.4%** | 顧客が「困っている」と言うかどうかは顧客次第。営業が言わせることはできない | 下位の方が多い→下位の顧客は課題が深刻（=ニーズはある）が、営業がそれを受注に変換できていない |
| **CP61_顧客前向き反応** | +3.2% | 「いいですね」と顧客が言うのは結果。営業が強制できない | 差がほぼない=前向き反応の有無では勝敗は決まらない |
| **CP62_顧客料金質問** | **-8.8%** | 顧客が「いくら？」と聞くのは顧客の判断 | **下位の方が多い→価値提案が弱いから料金にしか話が行かない。料金質問自体が悪いのではなく、それ以前のQ2が貧弱なことが問題** |
| **CP63_顧客懐疑反応** | +1.1% | 顧客が疑問を持つのは顧客次第 | 差がない=懐疑的な顧客は上位にも下位にも同じくらいいる |
| **CP64_顧客上申言及** | -2.5% | 「上に聞きます」は顧客の組織構造と意思決定の問題 | 差がない。ただし佐藤だめだめ期は+17pt多い→先送りの口実に使われている可能性 |

**→ 顧客反応系は「訓練対象」ではなく「ダッシュボード」。これらの数値が変われば、商談の質が変わった証拠。**

### ホスト側だが結果的に現れるもの

| CP | 統合差分 | なぜ相関か | 正しい読み方 |
|----|---------|-----------|-------------|
| **CP52_承知フレーズ** | +9.6% | 「承知しました」は意識すれば増やせるが、**良い対話のリズムがあれば自然に出る**。言い方を変えれば、顧客が発言している（=聞かせている）から「承知しました」が返せる | 上位は顧客を喋らせているから承知フレーズが増える。**鍛えるべきは「顧客を喋らせる力」であって「承知しましたの回数」ではない** |
| **CP51_共感フレーズ** | +5.4% | 同上。顧客の発言に対する反応だから、顧客が発言しなければ共感もできない | 同上 |
| **CP50_名前呼び** | -1.7% | ほぼ全員がやっている(97-98%) | 非差別化。基本マナーレベル |
| **CP14_具体数字** | -3.3% | 90%超で両方が使っている。数字を使うこと自体は差にならない | **数字の「量」ではなく「使い方」が問題。上位はデータとして使い、下位はコスト比較として使う** |

### 下位の方が多い項目の再解釈

| CP | 統合差分 | 従来の解釈 | 因果/相関を分けた解釈 |
|----|---------|-----------|---------------------|
| **CP22_コスト比較** | -8.6% | 下位はコスト比較をしすぎ | **相関**: 価値提案(Q2)が弱い → 顧客が料金に固執 → 営業もコスト比較で対応するしかない。**根本原因はQ2のフレーミング不足** |
| **CP42_期限設定** | -6.6% | 下位は期限設定をしすぎ | **相関**: クロージングの切り札がないから期限で押すしかない。**根本原因はQ2-Q3の説得力不足** |
| **CP62_顧客料金質問** | -8.8% | 料金を聞かれすぎ | **相関**: 上記と同じ構造。**Q2でフレーミングできていれば、顧客の質問は料金ではなく「具体的にどうやるの？」になるはず** |

## 3. 再構成: 真に鍛えるべき6項目 と 監視すべき指標

### 鍛えるべき（因果的・意図的行動）

```
優先度1: フレーミング力（Q2で勝敗が決まる）
  ├→ CP20_人事部リフレーム  — 視点変換のスクリプト
  ├→ CP23_他社事例         — 事前準備（事例カード）
  └→ CP12_求人データ       — 事前準備（データシート）

優先度2: 戦略的質問力（Q1で土台を作る）
  ├→ CP07_決裁確認         — 商談の早い段階で聞く
  ├→ CP04_紹介経験確認      — 顧客の経験値を測る
  └→ CP05_課題ヒアリング深さ — レベル3（因果的掘り下げ）まで到達する

以上6つ。これ以外は「やれ」と言って改善するものではない。
```

### 監視すべき（相関的・結果指標）

```
健全な商談の兆候（これが出ていればOK）:
  ├→ CP52_承知フレーズ多い   = 顧客が喋っている証拠
  ├→ CP51_共感フレーズ多い   = 対話が成立している証拠
  └→ CP61_顧客前向き反応あり  = 興味を持っている証拠

危険な商談の兆候（これが出たら上流を疑え）:
  ├→ CP62_顧客料金質問が早い  = Q2の価値提案が弱い
  ├→ CP22_コスト比較に時間    = フレーミングが刺さっていない
  ├→ CP64_顧客上申言及       = 決裁者に届いていない or 先送りの口実
  └→ CP60_顧客課題言語化なし  = ヒアリングが表面的
```

### 因果↔相関の構造図

```
【因果チェーン: Q2のフレーミングが起点】

CP20_人事部リフレーム ─┐
CP23_他社事例 ────────┤ ★意図的に実行する（因果）
CP12_求人データ ──────┘
        │
        ▼
  [顧客の視点が変わる]
        │
        ├→ CP62_顧客料金質問が減る      ← 相関（結果）
        ├→ CP22_コスト比較が減る        ← 相関（結果）
        ├→ CP61_顧客前向き反応が増える    ← 相関（結果）
        └→ CP52_承知フレーズが増える      ← 相関（結果）


【因果チェーン: Q1のヒアリングが起点】

CP07_決裁確認 ────────┐
CP04_紹介経験確認 ────┤ ★意図的に実行する（因果）
CP05_課題ヒアリング ──┘ ← ★深さが鍵（レベル3まで掘り下げる）
        │
        ▼
  [顧客のニーズ構造を把握]
        │
        ├→ CP60_顧客課題言語化が増える    ← 相関（結果）＝深いヒアリングの直接的な成果
        ├→ CP64_顧客上申言及が減る        ← 相関（決裁者を特定済みだから）
        ├→ CP63_顧客懐疑反応が減る        ← 相関（的を射た提案だから）
        └→ Q2フレーミングの命中率が上がる  ← 深い理解があるから刺さる
```

## 4. アクション設計の修正

### 修正前（相関を因果として扱っていた部分）

| 施策 | 問題 |
|------|------|
| 「承知フレーズを最低5回使え」 | ❌ 回数を増やしても意味がない。顧客が喋る環境を作れば自然に増える |
| 「名前呼びを意識的に多用」 | ❌ 97%が既にやっている。差別化要因ではない |
| 「コスト比較はQ3にまとめろ」 | △ 半分正しいが、本質はQ2のフレーミング不足。コスト比較を禁止するのではなく、その前にフレーミングを入れることが正解 |
| 「期限はキャンペーンとセットで」 | △ テクニック論。期限設定が多い=他に切り札がないことの方が問題 |

### 修正後（因果に絞った施策）

| 施策 | 対象 | 根拠 |
|------|------|------|
| **フレーミング3種のスクリプト整備 + ロープレ** | 全員 | CP20/23/12は因果的。意図して準備・実行するもの |
| **決裁確認・紹介経験確認のヒアリング追加** | 全員 | CP07/04は因果的。聞くか聞かないかの判断 |
| **商談前データ準備シート** | 全員 | データ根拠/分を支える因果的な準備行動 |
| **顧客料金質問の出現タイミングをモニタリング** | マネージャー | Q2前に出たら黄色信号。フレーミングが刺さっていない証拠 |
| **承知/共感フレーズの出現数をモニタリング** | マネージャー | 商談の健全性指標として追跡。低い場合は「営業が一方的に話している」兆候 |

### 佐藤への指示の修正

| 修正前 | 修正後 |
|--------|--------|
| 「承知フレーズを5回使え」 | **「顧客に3回以上質問し、返答に対して応答しろ」** → 結果として承知フレーズが自然に増える |
| 「示唆質問を1商談2回」 | **「データを出してから示唆質問しろ。データなしの示唆質問は禁止」** → 示唆質問の効果が復活する条件を指定 |
| 「名前呼びを多用」 | **削除**。名前呼びは差別化要因ではない |

## 5. まとめ: チェックポイントの最終分類

| 分類 | CP | 施策の方向 |
|------|-----|-----------|
| **因果 × 訓練対象** | CP20人事部リフレーム, CP23他社事例, CP12求人データ, CP07決裁確認, CP04紹介経験確認, **CP05課題ヒアリング(深さ)** | スクリプト化 + ロープレ + 準備シート + 3段階掘り下げ訓練 |
| **因果 × 基本動作** | CP03現状手法, CP24サービス説明, CP44プラン提示, CP41キャンペーン | やること自体は全員できている。質・タイミング・組み立ての問題 |
| **相関 × 健全性指標** | CP52承知, CP51共感, CP61顧客前向き | ダッシュボードとして監視。数値が上がれば商談の質が上がった証拠 |
| **相関 × 危険信号** | CP62顧客料金質問, CP22コスト比較, CP42期限設定 | 出現が早い/多い場合はQ2のフレーミング不足を疑う |
| **相関 × 顧客反応** | CP60課題言語化, CP63懐疑, CP64上申 | 顧客の状態を示す指標。直接コントロール不能 |

**最終的に「これをやれ」と言えるのは6つだけ。残りは全て温度計。**
