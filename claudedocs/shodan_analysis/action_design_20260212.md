# 商談力向上アクション設計書

**作成日**: 2026-02-12
**根拠データ**: 613件の商談文字起こし分析 + Salesforce Opportunity受注/失注データ（16名）
**分析期間**: 2024年前半 vs 2026年1月

---

## 0. エグゼクティブサマリー

### 分析から導かれた3つの事実

| No | 事実 | 根拠 |
|----|------|------|
| 1 | **「何を言うか」ではなく「どう組み立てるか」が差** | 2024H1で12個あった必須CP → 2026Janで0個。テクニック自体は均質化済み |
| 2 | **データ根拠/分が唯一の生存差別化要因** | 両期間でd>1.0（効果量大）、受注率相関r=+0.52 |
| 3 | **フレーミング（視点変換）ができる人が勝つ** | 統合分析で★必須：人事部リフレーム(+20.9%)、他社事例(+17.2%)、求人データ(+15.3%) |

### 因果 vs 相関の区別（重要）

34チェックポイントのうち、「これをやれ」と訓練可能なのは**因果的な5項目のみ**。
残りは商談の健全性を示す**温度計（相関指標）**であり、直接鍛える対象ではない。

詳細: `claudedocs/shodan_analysis/causal_vs_correlation_20260212.md`

| 分類 | 対象CP | 施策 |
|------|--------|------|
| **因果（訓練対象）** | CP20人事部リフレーム, CP23他社事例, CP12求人データ, CP07決裁確認, CP04紹介経験確認 | スクリプト化 + ロープレ + 準備シート |
| **相関（健全性指標）** | CP52承知, CP51共感, CP61顧客前向き | ダッシュボードで監視。上がれば商談の質が改善した証拠 |
| **相関（危険信号）** | CP62顧客料金質問, CP22コスト比較, CP42期限設定 | 出現が早い/多いならQ2のフレーミング不足 |

### アクション設計の方針

```
Phase 1: 全員底上げ（2週間）  → 因果的CPの意識化 + 準備シート導入
Phase 2: 差別化強化（4週間）  → フレーミング力とデータ活用力の強化
Phase 3: 個別最適化（継続）  → メンバー別弱点の重点改善 + 相関指標のモニタリング
```

## 1. Phase 1: 全員底上げ（Week 1-2）

### 1-1. 商談チェックシートの導入

以下のチェックシートを印刷/共有し、**商談後に自己採点**する。

#### 【商談チェックシート】（商談後5分で記入）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  商談チェックシート        日付:___  担当:___  相手:___
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

■ Q1（冒頭 0-25%）— ヒアリングフェーズ
  □ 職種を具体的に確認した（介護/看護/リハ等）
  □ 採用人数を聞いた（「何名くらいお探しですか？」）
  □ 現在の採用手法を聞いた（ハロワ/紹介/媒体）
  □ 課題・困りごとを質問した
  □ 「○○様」と名前で呼んだ（3回以上）
  □ 共感フレーズを使った（「おっしゃる通り」等）

■ Q2（中盤前半 25-50%）— データ提示フェーズ
  □ 競合施設のデータを提示した
  □ 調査データ/市場データを見せた
  ★□ 他社事例を伝えた（「同じエリアの○○法人では…」）

  □ 決裁プロセスを確認した

■ Q3（中盤後半 50-75%）— クロージングフェーズ
  □ テストクロージング（「いかがですか？」）した
  □ 具体プランを提示した（スタンダード/ライト等）
  ★□ 求人データを提示した（求人倍率、応募件数等）
  □ キャンペーン/期間限定を伝えた（該当時）

■ Q4（終盤 75-100%）— 確約フェーズ
  □ 次回の日程を設定した

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 運用ルール
- 商談直後にSlack/スプレッドシートに投稿
- 週次で全員の平均チェック数をランキング
- ★マーク3つの実施率を重点KPIとする

### 1-2. 勝ちパターン時系列の共有（15分の朝会テーマ）

Week 1の朝会で以下の「理想的な商談の流れ」を共有する。

```
【勝ち商談の4フェーズ構造】

Q1 (0-25%)  ヒアリング + ラポール
  ├→ 職種・人数・現状手法の確認
  ├→ 課題ヒアリング（「困っていること」を引き出す）
  ├→ 示唆質問（「このままだとどうなりますか？」）← 早期に投入
  └→ エリアデータの提示（話の導入に使う）

Q2 (25-50%)  データ爆弾 + フレーミング ← ★最重要フェーズ
  ├→ 競合データ「近隣の施設では…」
  ├→ 求人データ「この地域の求人倍率は…」
  ├→ 他社事例「同じ規模の○○法人では…」
  ├→ 人事部リフレーム「御社の人事部として…」
  └→ 決裁プロセスの確認

Q3 (50-75%)  クロージング
  ├→ テストクロージング「率直にいかがですか？」
  ├→ プラン提示（複数案）
  ├→ キャンペーン提示（該当時）
  └→ 顧客の料金質問に対応 ← 値段に行く前にQ2で価値を積め

Q4 (75-100%)  確約 + 上申支援
  ├→ 次回日程の設定
  └→ 「院長/理事長にご提案する資料をお送りします」
```

**ポイント**: Q2の密度が商談の成否を決める。ここで「データ×フレーミング」のコンボを打てるかが勝敗の分岐点。

## 2. Phase 2: 差別化強化（Week 3-6）

### 2-1. フレーミング力の強化（★最優先）

統合分析で「★必須」判定の3項目は、すべて**顧客の視点を変えるフレーミング**。

#### 施策A: フレーミング3種のトークスクリプト整備

**① 人事部リフレーム（統合差分+20.9%、最大の差別化要因）**

```
【背景】
上位の36.9%が使用 vs 下位の16.0%。佐藤イケイケ期は56.2%で使用。

【スクリプト例】
  「今、御社では○○様が採用もされながら現場のお仕事もされていますよね。
   弊社をご利用いただくと、いわば御社の人事部・採用チームのような役割を
   担わせていただくことになります。○○様が本業に集中できる環境を作る、
   というのが弊社のご提案です。」

【使用タイミング】
  Q2の中盤、コスト比較の前。価値の土台を作ってから価格に入る。

【注意】
  下位が多い「コスト比較」(下位84.4% vs 上位75.8%)に先に入ると、
  価格交渉に引き込まれる。まずフレームを変えてから数字を出す。
```

**② 他社事例（統合差分+17.2%）**

```
【背景】
上位の35.9%が使用 vs 下位の18.7%。佐藤イケイケ期は61.6%で使用。

【スクリプト例】
  「同じ○○市内で、御社と同じくらいの規模の法人様がいらっしゃるんですが、
   そちらでは昨年弊社をご利用いただいて、3ヶ月で介護職2名の採用に
   成功されています。」

【準備】
  商談前に同エリア・同規模の導入事例を1-2件用意しておく。
  → Salesforce Opportunityの受注案件から、
    同都道府県×同サービスタイプで検索して事例ストックを作る。

【使用タイミング】
  Q2の競合データ提示の直後。「データ→事例→リフレーム」の三連打が理想。
```

**③ 求人データ提示（統合差分+15.3%）**

```
【背景】
上位の46.0%が使用 vs 下位の30.7%。佐藤イケイケ期は68.5%で使用。

【スクリプト例】
  「○○市の介護職の有効求人倍率は現在△.△倍でして、
   求職者1人に対して△社が奪い合っている状態です。
   この状況でハローワークだけで待っていると、
   正直かなり厳しいというのが市場のデータです。」

【準備】
  ハローワークの地域別有効求人倍率データを毎月更新し、
  商談相手のエリアの数字を事前にメモしておく。
```

### 2-3. 「やりすぎ注意」項目の是正

統合分析で「△下位が多い」と判定された項目は、**やっていること自体は悪くないが、タイミングや比重が間違っている**可能性を示す。

| 項目 | 下位が多い理由の仮説 | 是正アクション |
|------|---------------------|---------------|
| **顧客料金質問**（下位50.2% vs 上位41.4%） | Q2での価値提案が弱いため、顧客が早期に「で、いくら？」と聞く | Q2でフレーミング3種を徹底し、**顧客から料金を聞かれる前に価値を積む** |
| **コスト比較**（下位84.4% vs 上位75.8%） | 自らコスト比較に時間を使いすぎ | コスト比較は**Q3のプラン提示時にまとめて**。Q2では価値提案に集中 |
| **期限設定**（下位36.9% vs 上位30.3%） | 期限だけ設定しても効果薄 | 期限は**キャンペーンとセットで**のみ使用 |

### 3-2. 佐藤丈太郎 専用リカバリープログラム

佐藤の問題は**単一のスキル不足ではなく、5層の複合崩壊**（前回分析参照）。段階的に再構築する。

#### Step 1: データストックの再構築（Week 1-2）

| 施策 | 内容 | 目標値 |
|------|------|--------|
| 事例カード作成 | SF Opportunityから過去受注30件の事例カードを作成 | 30枚 |
| エリアデータ整備 | 担当エリアの求人倍率・人口・競合施設を一覧化 | 担当エリア全域 |
| 準備シート必須化 | 全商談で商談前データ準備シートを記入してからコール | 100% |
| 目標KPI | データ根拠/分 | 0.311 → 0.5（まず全体平均に追いつく） |

#### Step 2: 示唆質問の再配置（Week 3-4）

佐藤のイケイケ期は示唆質問91.8%だったが、だめだめ期は31.2%に激減。
ただし、**示唆質問は「データ根拠とセットで使わないと機能しない」**ことが判明（2026JanでSPIN Iの効果量がd=0.07に消失）。

| 施策 | 内容 |
|------|------|
| 示唆質問の「前振り」を変える | データを出す → 示唆質問する、の順序を徹底。「○○市の求人倍率は△倍です。このままハローワークだけで待ち続けると、正直どうなると思いますか？」 |
| 示唆質問は1商談2回まで | やりすぎると詰問的。Q1で1回、Q3で1回が適切 |
| Q1の課題ヒアリングを復活 | イケイケ期は課題質問90.4% → だめだめ期71.9%に低下。冒頭で「今一番お困りのことは何ですか？」を必ず入れる |

#### Step 3: 承知フレーズの回復（Week 3-4）

佐藤の最大の行動変化は**承知フレーズの激減（98.6% → 43.8%、-54.8pt）**。
丁寧さの崩壊が顧客との信頼構築を阻害している可能性。

| 施策 | 内容 |
|------|------|
| 意識的な「承知しました」の挿入 | 顧客の発言に対して「承知しました」「かしこまりました」を最低5回使う |
| 名前呼びの回復 | 2.46/分 → 1.44/分（-41.4%）に低下。「○○様」を意識的に多用 |

#### Step 4: フレーミングの再習得（Week 5-6）

| 他社事例 | 61.6% | 12.5% | 50%以上 |

## 4. KPI設計

## 7. 注意事項・リスク

### 7-1. 「テクニックのコモディティ化」リスク

2024H1→2026Janで起きた「差別化要因の消失」は、チーム全体がテクニックを学んだ結果。
同じことがPhase 1-2の施策でも起きうる。

**対策**: Phase 3の個別最適化を重視し、**「何を言うか」ではなく「商談全体の構成力」**を育てる。チェックシートは入口であり、最終的には自然にフレームを組み立てられる状態を目指す。

### 7-2. PI比率の過剰是正リスク

篠木・市来のPI比率を下げる指導をする際、**質問すること自体を否定しない**よう注意。
問題は質問の量ではなく、「詰問的に感じさせる質問の連続」。
データ提示 → 質問 → 共感 → データ提示 のサイクルに変えることが目標。

### 7-3. 佐藤のメンタルケア

佐藤は8.3%→0.0%という急激な成績低下を経験している。改善プログラムを「叱責」ではなく「再構築」として位置づけ、小さな成功体験（チェックシートの★実施率向上、データ根拠/分の改善など）を承認しながら進める。
