## 1. 法人格推論の概要と精度検証

### 1.1 背景と目的

Salesforceの `Account.LegalPersonality__c`（法人格）フィールドは全Accountの約62%にしか入力されておらず、セグメント分析の精度に影響していた。国税庁の法人番号公表CSVを用いて、法人番号 → 法人名 → 正規表現による法人格抽出 という推論パイプラインを構築し、法人格のカバレッジを向上させた。

### 1.2 推論パイプライン

```
Salesforce Account.CorporateNumber__c
    ↓ 突合
国税庁 法人番号公表CSV（約500万法人）
    ↓ マッチ
法人名を取得
    ↓ 正規表現抽出
法人格を推論（株式会社, 医療法人, 社会福祉法人, etc.）
```

**推論対象**: 47,411件（法人格未入力かつ法人番号ありのAccount）

### 1.3 精度検証結果

#### 検証1: 法人名と推論法人格の整合性

| 指標 | 結果 |
|------|------|
| 整合 | 47,411件 / 47,411件 |
| **整合率** | **100.00%** |

推論された法人格が国税庁の法人名に必ず含まれていることを確認。

#### 検証2: ベンチマーク（法人格既存Accountでの検証）

法人格が既に入力されている213,556件のAccountに対し、同じ推論ロジックを適用して精度を測定。

| 指標 | 結果 |
|------|------|
| ベンチマーク対象 | 204,704件（国税庁CSVマッチ分） |
| 既存値と推論一致 | 198,123件 |
| 不一致 | 6,581件 |
| **精度** | **96.8%** |

#### 検証3: 不一致の原因分析

不一致6,581件を詳細分析した結果、**Salesforce側の法人番号データ品質の問題**が主因と判明。

- Account名から法人格を推論可能だった3,017件のうち、**99.7%**（3,008件）で「Account名推論 = 既存値」
- NTA推論（法人番号経由）と一致したのはわずか4件（0.1%）
- つまり、法人番号が誤ったレコードを指している

**不一致パターンの主要な遷移マトリクス（上位）:**

| 既存値 → NTA推論 | 件数 |
|----------------|------|
| 医療法人 → 株式会社 | 671件 |
| 株式会社 → 社会福祉法人 | 549件 |
| 社会福祉法人 → 株式会社 | 641件 |
| 社会福祉法人 → 医療法人 | 295件 |
| 医療法人 → 社会福祉法人 | 390件 |

#### 検証4: 名前不一致の背景

Account名とNTA法人名の一致率は18.9%と低いが、これは介護業界の特性（施設名 ≠ 法人名）による正常な現象。

| パターン | 件数 | 割合 |
|---------|------|------|
| 完全一致 | 8件 | 0.0% |
| 法人格の差のみ | 1,629件 | 3.4% |
| 部分重複あり | 7,312件 | 15.4% |
| 3文字以上部分一致 | 4,124件 | 8.7% |
| 完全に異なる | 34,338件 | **72.4%** |

完全に異なる34,338件のうち9,298件は複数事業所を持つ法人の施設（例: 社会福祉法人高佳会 → 馬瀬グループホームいきいき, デイサービスセンターいきいき）。

## 6. 発見事項とビジネスインサイト

### 6.1 法人格推論の有効性

- 法人格カバレッジを62.4% → 86.1%に改善（+7,505件）
- 推論精度96.8%（ベンチマーク検証済み）

### 6.2 高成約率セグメント

**最注力セグメント（ボリューム × 成約率のバランス）:**

1. **介護（高齢者） × 株式会社**（1,537件, 15.9%） - 最大の収益機会
2. **介護（高齢者） × 社会福祉法人**（1,306件, 12.9%） - 2番目に大きい
3. **障がい福祉 × 株式会社**（384件, 16.9%） - 高成約率
4. **障がい福祉 × 社会福祉法人**（204件, 24.0%） - 最高クラスの成約率
5. **保育 × 株式会社**（131件, 17.6%） - 高成約率
6. **保育 × 社会福祉法人**（316件, 14.6%） - ボリュームあり

### 6.3 医療法人の規模別傾向

医療法人は規模が成約率に大きく影響:

| 従業員規模 | 商談 | 成約率 | 示唆 |
|-----------|------|--------|------|
| 小規模(30以下) | 523 | **8.2%** | 個人クリニック・小規模医院 → 狙い目 |
| 不明 | 634 | 7.1% | - |
| 中規模(31-100) | 859 | 5.4% | 中規模病院 |
| 大規模(101+) | 2,075 | **4.3%** | 大病院 → 営業効率が低い |

**示唆: 医療法人は小規模（30人以下）に集中すべき**

### 6.4 FacilityType_Large__c フロー変更の影響

- 2025-06以降のデータでは施設形態別の分析精度が大幅に低下
- 2025-06以降に FacilityType_Large__c が入力されているのは成約商談のほぼ全量
- **フロー修復を推奨**: 全商談に対して施設形態が入力されるフローに戻すべき

### 6.5 「不明」セグメントの特異な挙動

法人格「不明」の成約率はFY2024で1.9%と低いが、FY2025では6.8%に上昇。これは2025-06以降の「介護」→null化による見かけ上の変化と推測される。

## 付録: 分析上の注意事項

1. **FacilityType_Large__c の「介護」値について**: Salesforce上に「介護」という値は存在しない。実際の値は「介護（高齢者）」。分析スクリプトで ServiceType/IndustryCategory から補完した際に「介護」として分類されたもの。
2. **2025-06以降のデータ**: FacilityType_Large__c のフロー変更により、施設形態別分析の信頼性が低下。全体集計（法人格のみ、法人格×従業員規模）は影響を受けない。
3. **Salesforce法人番号の品質**: 全体の3.2%で法人番号が誤ったレコードを指している。推論結果を Salesforce に反映する場合は DISAGREE 28件を除外すること。
4. **IsWon フィールド**: 成約判定は `IsWon = true` を使用（StageName は「受注」であり「成約」ではない）。
